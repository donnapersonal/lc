# 80.Remove Duplicates from Sorted Array II

难度：Medium 
标签: Array、Two Pointers

## 题目

![80](https://github.com/donnapersonal/picx-images-hosting/raw/master/image.232j9m1oxx.webp)

Constraints:
- 1 <= nums.length <= 3 * 10^4
- -10^4 <= nums[i] <= 10^4
- nums is sorted in non-decreasing order.

## 解题

### Brute Force**
> - 使用一个额外的数组
> - 遍历 `nums`
> - 统计当前数字出现的次数
> - 如果出现次数小于或等于 2，将其添加到新的结果数组中
> - 将结果复制回 `nums`
> - 返回结果数组的长度
>
> 时间复杂度：`O(n)`，空间复杂度：`O(n)`
> - 该方法可行，但不满足理想的“原地（in-place）”操作要求
>
> 原地操作
> - 遍历数组，统计当前值连续出现的次数。
> - 如果某个元素出现超过两次：
>   - 将后续所有元素向左移动一位。
>   - 减小有效长度。
>   - 不移动索引 `i`；重新检查当前位置
>
> 时间复杂度：`O(n²)`，空间复杂度：`O(1)`
> - 每次删除操作都可能涉及移动后续的 `O(n)` 个元素
>
> 瓶颈：
> - 每发现一个多余的重复元素，都要将后续所有元素向左移动
> - [1,1,1,1,1] -> 这种情况需要反复移动大量元素
>
> 为什么删除后不执行 i++？
> - 例如：
> [1,1,1,1,2]
>      i
>
> 删除索引 2 后：
> 
> [1,1,1,2,_]
>      i
> 
> 删除一个 1 后：
> 
> [1,1,2,_]
>    ↑
> 
> 新的 nums[i] 仍然可能是第三个 1，所以必须继续检查当前位置

```python
class Solution:
    def removeDuplicates(self, nums: List[int]) -> int:
        res = []
        for num in nums:
            if len(res) < 2 or res[-2] != num:
                res.append(num)
   
        nums[:] = res
        # for i in range(len(res)):
        #     res[i] = nums[i]
        return len(res)

# In-place modify
class Solution:
    def removeDuplicates(self, nums: List[int]) -> int:
        n = len(nums)
        i = 2

        while i < n:
            if nums[i] == nums[i - 1] == nums[i - 2]:
                for j in range(i, n - 1):
                    nums[j] = nums[j + 1]

                n -= 1
            else:
                i += 1

        return n
```

### 最优解
> 由于数组已排序，重复元素是相邻的。我们需要确保每个数字最多保留两次。
> - 无需显式计数重复次数，我们可以利用这一规则：`if nums[slow-2] != nums[fast]:`
> - 我维护一个有效的“结果前缀”：`nums[0 : slow]`
>   - `slow` 指针表示：下一个可以写入有效数字的位置
> - 由于每个数字最多出现两次，前两个元素总是可以安全保留的
>
> 因此，初始状态为：`slow=2`，`fast=2`
> - 对于每个 `nums[fast]`，我们思考：如果把这个数字放入 `nums[slow]`，是否会导致同一个数字出现三次？
> - 为了判断这一点，我将 `nums[fast]` 与写入位置前两个位置的元素（即 `nums[slow-2]`）进行比较
> - 如果 `nums[slow-2] == nums[fast]`，说明有效前缀的末尾已经包含了两个该数值
> - 再加入一个就会导致出现三次，因此我们跳过它
> - 如果 `nums[slow-2] != nums[fast]`，则加入 `nums[fast]` 是安全的
>
> 循环不变量 (Invariant)
> - 在每次迭代之前，`nums[0...slow-1]` 包含了所有已处理的元素，且没有任何数值出现超过两次
>
> 为什么需要 `fast` 和 `slow` 两个指针？
> - 我使用 `fast` 指针遍历数组，用 `slow` 指针标记下一个有效元素应放置的位置
> - 这样可以在原地进行更新，同时保持有效元素的顺序

**Solution**
> - 我使用慢指针（slow pointer）来表示有效前缀的下一个写入位置。前两个​​元素总是有效的，因此我从索引 2 开始设置 `slow`
> - 然后，我使用快指针（fast pointer）从索引 2 开始扫描数组。对于每个 `nums[fast]`，将其与 `nums[slow - 2]` 进行比较。 >   - 如果两者不同，说明加入 `nums[fast]` 不会产生三个相同的数字，因此将其写入 `nums[slow]` 并将 `slow` 加 1
>   - 如果两者相同，说明有效前缀中已包含该值的两个副本，因此跳过它
> - 最后，`slow` 即为有效前缀的长度

```python
class Solution:
    def removeDuplicates(self, nums: List[int]) -> int:
        # Get the size of the array. I'll use it to control our loop boundaries.
        n = len(nums)
        # If the array has 0 or 1 element, no duplicates can exist. 
        # I return the array length as is.
        if n < 2:
            return n
        
        # slow point to the next position to write a valid number.
        # fast scans through the array starting from index 2.
        # Why 2? 
        # - Because the first two elements are always valid — we allow up to 2 of each.
        slow, fast = 2, 2

        # Loop through all remaining elements starting from index 2.
        while fast < n:
            # We check if the number at fast is different from the number at slow - 2
            # Why slow - 2? It represents the second-to-last placed number
            # This check ensures we don't allow more than two of the same number
            if nums[slow-2] != nums[fast]:
                # If it's allowed, write the number at slow position.
                nums[slow] = nums[fast]
                # Move the write pointer forward.
                slow += 1
            
            # Always move the read pointer forward.
            fast += 1
        
        # After the loop, slow points to the end of the valid portion of the array. So the new length is slow.
        return slow

class Solution:
    def removeDuplicates(self, nums: List[int]) -> int:
        # Store array length to avoid recomputing it.
        n = len(nums)
        # Corner case: empty array → result is 0.
        if n == 0:
            return 0
        
        # fast: iterates through the array
        # slow: position to place the next valid element
        # everything from nums[0]..nums[slow] is guaranteed valid
        slow, fast = 0, 0
        
        # Tracks how many times the current number has appeared so far.
        # For example, for [1,1,1,1], count will go 0→1→2→3…
        # We only accept numbers while count < 2.
        count = 0
        
        # Scan all elements using fast.
        while fast < n:
            # The value changes → new number group starts
            # Now we will start counting occurrences of this new value.
            if nums[fast] != nums[slow]:
                # Move write pointer to next position
                slow += 1
                # Place this new number
                nums[slow] = nums[fast]
            # Same value as previous but used less than twice
            # We check:
            # slow < fast: ensures we’re not comparing the same index
            # count < 2: only allow max 2 occurrences
            elif slow < fast and count < 2:
                slow += 1
                nums[slow] = nums[fast]
            
            # Always move the scanning pointer
            fast += 1
            # We have consumed one more element of this number
            count += 1
            
            # Meaning:
            # When the next number differs from the current number
            # We reset count to 0
            # Because next element starts a new group
            # This logic ensures that for each number group we track how many have been accepted so far.
            if fast < n and nums[fast] != nums[fast - 1]:
                count = 0
        
        # Since slow is an index (0‑based), number of valid elements is slow + 1.
        return slow + 1
```

```java
class Solution {
    public int removeDuplicates(int[] nums) {
        int n = nums.length;
        if (n < 2) {
            return n;
        }

        int slow = 2;
        int fast = 2;
        while (fast < n) {
            if (nums[slow-2] != nums[fast]) {
                nums[slow] = nums[fast];
                slow ++;
            }
            fast ++;
        }
        return slow;
    }
}
```

**Complexity**
> - Time: `O(n)`, 每个元素处理最多 1 次
> - Space: `O(1)`, 没有使用额外空间
