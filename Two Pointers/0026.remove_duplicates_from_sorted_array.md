# 26.Remove Duplicates from Sorted Array

难度：Easy  
标签: Array、Two Pointers

## 题目

![26](https://github.com/donnapersonal/picx-images-hosting/raw/master/image.6bhqjc1u04.webp)

Constraints:
- 1 <= nums.length <= 3 * 10^4
- -100 <= nums[i] <= 100
- nums is sorted in non-decreasing order.

## 解题

### Brute Force
> - 发现相邻元素时，将后续所有元素向左移动一个位置
> - 随后减小数组有效长度
>
> Time: `O(n^2)`, Space: `O(1)`
> - 最坏情况下，需要移动大量元素
>
> Bottleneck:
> - 重复移动元素
> - [1,1,1,1,1] -> 这个需要大量重复移动元素
>
> 为什么删除后不执行 i++？
> - 因为新元素被移动到了 nums[i]，需要再次检查：
> 
>   [1,1,1,2]
>      ↑
> 
> 删除一个 1 后：
> 
> [1,1,2,_]
>    ↑
> 
> 当前位置仍然可能重复

```python
class Solution:
    def removeDuplicates(self, nums: List[int]) -> int:
        n = len(nums)
        i = 1
        while i < n:
            if nums[i] == nums[i-1]:
                for j in range(i, n-1):
                    nums[j] = nums[j+1]
                
                n -= 1
            else:
                i += 1
        
        return n
```

### 最优解

> 只需维护下一个应写入新唯一值的位置
>
> 核心思想：
> - 使用双指针：
>   - `read`（读指针） -> 遍历 `nums` 以寻找新的唯一元素
>   - `write`（写指针） -> 指向下一个应写入新唯一值的位置
> - 若 `nums[read] != nums[write - 1]` -> 说明 `nums[read]` 是一个新的唯一元素，将其放入 `nums[write]` 处
>
> 为什么 two pointes?
> - 因为数组是有序的，所有重复元素是相邻的， 使用双指针：
>   - `fast` 用来扫描数组来寻找新的元素
>   - `slow` 用于追踪下一个唯一值应放置的位置
> - 这使我们能够通过单次遍历，原地覆盖重复项
>
> 为什么 write = 1？
> - 如果数组非空，第一个元素一定是唯一元素，因此：nums[0] 已经属于有效区域
> - 下一个唯一元素应该写入索引 `1`
>
> Invariant｜循环不变量
> - 在每次迭代之前，nums[0...slow-1] 包含了迄今为止所见到的每个不同值，且每个值仅出现一次

**Solution**
> - if input is empty, return 0
> - initialize slow = 1, fast start from 1
> - compare the current element with the last retained element
> - if they are different, write the current element to nums[slow]
> - slow += 1
> - return slow
>

```python
class Solution:
    def removeDuplicates(self, nums: List[int]) -> int:
        # Early return: If the input list is empty, there are no elements to process, so return 0 immediately.
        if not nums:
            return 0
        
        # Stores the length of the array in variable n for convenience.
        n = len(nums)
        # Initializes two pointers:
        # slow: Points to the next position where a unique value should be placed.
        # fast: Iterates through the array to find new unique values.
        # Both start from index 1 since the first element is always unique.
        slow = fast = 1

        # Loop until the fast pointer reaches the end of the array.
        while fast < n:
            # Compares the current element with the previous one.
            # Since the array is sorted, a new value will differ from its previous one.
            # This condition filters out duplicates.
            if nums[fast] != nums[fast - 1]:
                # Places the unique value found by fast into the slow pointer’s position.
                # This "compacts" all unique elements at the front of the array.
                nums[slow] = nums[fast]
                # Moves slow forward, preparing for the next unique value.
                slow += 1
            
            # Regardless of whether we found a duplicate or not, fast always moves to the next element.
            fast += 1
        
        # slow now represents the count of unique elements found.
        # All unique values are stored in nums[0:slow].
        return slow

# 写法 2
class Solution:
    def removeDuplicates(self, nums: List[int]) -> int:
        n = len(nums)
        if n < 1:
            return n
        
        slow, fast = 0, 0

        while fast < n:
            if nums[fast] != nums[slow]:
                slow += 1
                nums[slow] = nums[fast]

            fast += 1

        return slow + 1 
```
```java
class Solution {
    public int removeDuplicates(int[] nums) {
        int n = nums.length;
        if (n == 0) {
            return 0;
        }

        int slow = 1;
        int fast = 1;
        while (fast < n) {
            if (nums[fast] != nums[fast-1]) {
                nums[slow] = nums[fast];
                slow ++;
            }
            fast ++;
        }
        return slow;
    }
}
```

**Complexity**
> - Time: `O(n)`, each element is processed at most once
> - Space: `O(1)`, no extra space
