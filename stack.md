
# 规律

## 什么会被归到 Stack / Monotonic Stack？

核心判断一句话：
- 只要题目需要处理“最近的还没解决的东西”，就很可能用 `Stack`
- 只要题目需要找“左边/右边第一个更大或更小”，就很可能用 `Monotonic Stack`

## 普通 Stack 的规律

普通栈解决的是：`后出现的东西，先被处理`，也就是 `Last In First Out`

常见场景有 `4` 类

**1. 处理嵌套结构**

看到这些关键词，就想 `Stack`：
- 括号
- 路径
- 表达式
- 编码字符串
- 函数调用
- 树的 preorder 恢复

对应题：

| 题目                                 | 为什么用 Stack                   |
| ---------------------------------- | ---------------------------- |
| 394. Decode String                 | `3[a2[c]]` 有嵌套，遇到 `]` 要回到上一层 |
| 636. Exclusive Time of Functions   | 函数调用天然是嵌套的，当前运行函数一定是栈顶       |
| 1028. Recover a Tree From Preorder | 根据深度恢复树，最近的父节点在栈顶            |
| 71. Simplify Path                  | 遇到 `..` 要回退到上一个目录            |

通俗理解：

嵌套结构就像：

外层
  内层
    更内层

必须先处理最里面，再回到外面 -> 这就是`栈`

**2. 处理表达式计算**

对应题：

| 题目                                    | 为什么用 Stack          |
| ------------------------------------- | ------------------- |
| 150. Evaluate Reverse Polish Notation | 遇到运算符时，取最近两个数字计算    |
| 227. Basic Calculator II              | 乘除优先级高，需要回头修改最近一个数字 |

通俗理解：

表达式里经常要处理：
- 最近的两个数
- 最近的一个操作符
- 最近的一段表达式

所以用`栈`保存之前还没最终确定的结果

**3. 处理碰撞 / 删除 / 抵消**

对应题：

| 题目                                      | 为什么用 Stack               |
| --------------------------------------- | ------------------------ |
| 735. Asteroid Collision                 | 当前小行星只会和左边最近的右向小行星碰撞     |
| 1209. Remove All Adjacent Duplicates II | 当前字符可能和栈顶字符合并，达到 k 个就删除  |
| 402. Remove K Digits                    | 当前数字可以让前面较大的数字被删除        |
| 316. Remove Duplicate Letters           | 当前字符可能让前面较大的字符出栈，得到更小字典序 |

通俗理解

这些题都有一个特点：`当前元素来了以后，只关心它前面最近的一批元素`

比如小行星碰撞：[5, 10, -5]，-5 不会直接和 5 碰，它先和最近的 10 碰

所以用`栈`

**4. 设计类 / 迭代类题目**

对应题：

| 题目                                            | 为什么用 Stack                  |
| --------------------------------------------- | --------------------------- |
| 155. Min Stack                                | 除了正常栈，还要维护当前最小值             |
| 1381. Design a Stack With Increment Operation | 本质是设计一个支持特殊操作的栈             |
| 173. Binary Search Tree Iterator              | 用栈模拟中序遍历                    |
| 445. Add Two Numbers II                       | 链表从高位到低位，但加法要从低位开始，所以用栈反转顺序 |

通俗理解

如果题目要求你：`模拟某种“先存起来，后面倒着处理”的过程`

就很可能用`栈`

## Monotonic Stack 的规律

单调栈解决的是：`找某个元素左边或右边第一个比它大 / 小的元素`

常见关键词：

| 关键词              | 想到什么 |
| ---------------- | ---- |
| next greater     | 单调栈  |
| next smaller     | 单调栈  |
| previous greater | 单调栈  |
| previous smaller | 单调栈  |
| 子数组最小值           | 单调栈  |
| 子数组最大值           | 单调栈  |
| 最大矩形 / 最大宽度      | 单调栈  |
| 字典序最小            | 单调栈  |

**1. Next Greater 类**

对应题：

| 题目                                     | 本质            |
| -------------------------------------- | ------------- |
| 739. Daily Temperatures                | 找右边第一个更高温度    |
| 503. Next Greater Element II           | 找循环数组右边第一个更大值 |
| 1019. Next Greater Node In Linked List | 找链表中右边第一个更大节点 |

为什么用单调栈？

比如：[73, 74, 75, 71, 69, 72, 76]

当看到 72 时，它可以解决前面的：71 -> 72, 69 -> 72

栈里保存的是：`还没找到更大值的元素`

如果当前值比栈顶大，说明当前值就是栈顶的答案

所以：当前值 > 栈顶值 => 栈顶找到 next greater => pop

**三、子数组贡献类为什么用单调栈？**

对应题：

| 题目                                 | 本质                |
| ---------------------------------- | ----------------- |
| 907. Sum of Subarray Minimums      | 每个数作为最小值能贡献多少次    |
| 1856. Maximum Subarray Min-Product | 每个数作为最小值时，能控制多大范围 |

`907. Sum of Subarray Minimums`

题目要算：所有子数组的最小值之和

暴力会枚举所有子数组，太慢

换个角度：`不枚举子数组，而是枚举每个元素作为最小值的贡献`

对于 `arr[i]`，要知道它作为最小值能管多大范围，需要找：

- 左边第一个比它小的元素
- 右边第一个比它小的元素

这正是`单调栈`最擅长的

贡献公式：`arr[i] * 左边可选数量 * 右边可选数量`

`1856. Maximum Subarray Min-Product`

题目要最大化：子数组最小值 * 子数组和

如果固定 nums[i] 是最小值，就要找它能作为最小值的最大范围

所以需要：
- 左边第一个小于它的位置
- 右边第一个小于它的位置

然后用`前缀和`求这个范围的 `sum`

所以是：`单调栈 + 前缀和`

**四、字典序 / 删除类为什么也是单调栈？**

对应题：

| 题目                            | 本质              |
| ----------------------------- | --------------- |
| 402. Remove K Digits          | 删除 k 个数字，使结果最小  |
| 316. Remove Duplicate Letters | 保留每个字符一次，使字典序最小 |

`402. Remove K Digits`

比如：1432219, k = 3

如果当前数字比栈顶小，比如看到 3，前面有 4：4 > 3

为了让数字更小，应该删掉前面的 4

所以规则是：`只要当前数字更小，并且还能删除，就弹出栈顶较大的数字`

这就是`单调递增栈`

`316. Remove Duplicate Letters`

目标是：每个字母保留一次，且字典序最小

如果当前字符比栈顶小，且栈顶字符后面还会出现，那么可以弹出栈顶

比如：bcabc

遇到 a 时，前面的 b、c 如果后面还能出现，就可以弹掉，让结果更小

本质也是：`尽量保持栈内字符字典序递增`

**特殊单调栈题**

`456. 132 Pattern`

要找：nums[i] < nums[k] < nums[j], i < j < k

这是一个很典型的`“维护候选关系”`的单调栈题

从右往左扫：
- 栈维护可能的 `nums[j]`
- 变量维护可能的 `nums[k]`
- 如果当前 `nums[i] < nums[k]`，说明找到 `132 pattern`

本质还是：用单调栈维护右边可用的候选值

`962. Maximum Width Ramp`

题目要找：i < j 且 nums[i] <= nums[j], 最大 j - i

为什么用单调栈？
- 只需要保留“有可能成为左端点”的位置
- 如果前面某个位置的值更大，它不适合作为左端点，因为更小的值更容易满足：`nums[i] <= nums[j]`

所以先建立一个单调递减栈，保存优秀左端点

然后从右往左找能匹配的最大宽度

## 快速判断模板

**普通 Stack**

看到这些，就想普通栈：
- 最近的
- 嵌套的
- 配对的
- 撤销的
- 回退的
- 表达式
- 括号
- 路径
- 函数调用

典型逻辑：
- `遇到普通元素：push`
- `遇到结束 / 冲突 / 回退：pop`

**Monotonic Stack**

看到这些，就想单调栈：
- 右边第一个更大
- 右边第一个更小
- 左边第一个更大
- 左边第一个更小
- 子数组最小值
- 子数组最大值
- 字典序最小
- 删除若干元素后最优

典型逻辑：

```js
while stack and 当前元素能解决栈顶:
    pop 栈顶并结算答案

push 当前元素
```

## 最核心记忆

可以记这三句话：
- 普通栈：解决最近的未完成问题
- 单调栈：解决第一个更大/更小的问题
- 贡献类单调栈：解决每个元素能控制多大范围的问题

面试时只要你能说出：

“I use a stack because the current element only interacts with the most recent unresolved element.”

或：

“I use a monotonic stack because we need to find the nearest greater or smaller element efficiently.”
