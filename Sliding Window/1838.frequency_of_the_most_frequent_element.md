# 1838.最高频元素的频数

题目链接：[1838.最高频元素的频数](https://leetcode.cn/problems/frequency-of-the-most-frequent-element/)

## 题目大意

元素的 频数 是该元素在一个数组中出现的次数

给你一个整数数组 nums 和一个整数 k，在一步操作中，你可以选择 nums 的一个下标，并将该下标对应元素的值增加 1 

执行最多 k 次操作后，返回数组中最高频元素的 最大可能频数 

```js
Example 1:
Input: nums = [1,2,4], k = 5
Output: 3
Explanation: Increment the first element three times and the second element two times to make nums = [4,4,4].
4 has a frequency of 3.

Example 2:
Input: nums = [1,4,8,13], k = 5
Output: 2
Explanation: There are multiple optimal solutions:
- Increment the first element three times to make nums = [4,4,8,13]. 4 has a frequency of 2.
- Increment the second element four times to make nums = [1,8,8,13]. 8 has a frequency of 2.
- Increment the third element five times to make nums = [1,4,13,13]. 13 has a frequency of 2.

Example 3:
Input: nums = [3,9,6], k = 2
Output: 1
```

限制：
- 1 <= nums.length <= 10^5
- 1 <= nums[i] <= 10^5
- 1 <= k <= 10^5

## 解题

`排序+滑动窗口`

比如 `nums=[1,4,1,2]`，把其中的 `1,1,2` 都变成 `3` 是没有意义的，因为都变成 `2` 需要的操作次数更少，更容易实现

所以最高频元素一定可以等于某个 `nums[i]`

枚举这个数，设其为 `x`
- 由于只能把元素变大，所以只能把小于 `x` 的数变大
- 为了能操作更多的元素，优先把与 `x` 相差小的数变大

怎么快速找到要变大的数？
- 既然只关心元素的个数，并不关心元素的相对顺序，所以先把 `nums` 从小到大排序
- 排序后，要变大的数就是一段连续的子数组了。子数组越长，需要的操作次数越多，越无法实现；子数组越短，需要的操作次数越少，越可以实现。有这样的性质，可以用`滑动窗口`解决

设子数组的下标范围为 [left,right]，子数组和为 sum，那么需要操作多少次？

![alt text](https://github.com/donnapersonal/picx-images-hosting/raw/master/image.4xv5d1cn3t.webp)

如上图，矩形底边长为 `right−left+1`，高为 `x=nums[right]`，面积为 `(right−left+1)⋅x`，减去子数组和 `sum`，得到需要的操作次数（蓝色面积）为 `(right−left+1)⋅x−sum`

如果上式大于 `k`，需要缩短窗口，减少 `sum`，增加 `left`

缩短窗口到可以操作，用子数组长度 `right−left+1` 更新答案的最大值


```js
var maxFrequency = function(nums, k) {
    nums.sort((a, b) => a - b);
    let sum = 0, left = 0, res = 0;
    for (let right = 0; right < nums.length; right++) {
        const x = nums[right];
        sum += x;
        while ((right - left + 1) * x - sum > k) {
            sum -= nums[left];
            left++;
        }
        res = Math.max(res, right - left + 1);
    }
    return res;
};
```
```python
class Solution:
    def maxFrequency(self, nums: List[int], k: int) -> int:
        nums.sort()
        res = s = left = 0
        for right, x in enumerate(nums):
            s += x
            while (right - left + 1) * x - s > k:
                s -= nums[left]
                left += 1
            res = max(res, right - left + 1)
        return res
```
```java
class Solution {
    public int maxFrequency(int[] nums, int k) {
        Arrays.sort(nums);

        int left = 0;
        int res = 0;
        long windowSum = 0;

        for (int right = 0; right < nums.length; right++) {
            windowSum += nums[right];

            while ((long) nums[right] * (right - left + 1) - windowSum > k) {
                windowSum -= nums[left];
                left++;
            }

            res = Math.max(res, right - left + 1);
        }

        return res;
    }
}
```

- 时间复杂度：`O(nlogn)`，其中 `n` 是数组 `nums` 的长度。排序数组的时间复杂度为 `O(nlogn)`，使用滑动窗口遍历目标值的时间复杂度为 `O(n)`
- 空间复杂度：
  - `O(logn)`，即为排序数组需要使用的栈空间
  - `O(1)`，忽略排序的栈开销
