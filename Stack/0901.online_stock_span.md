# 901.在线库存跨度

题目链接：[901.在线库存跨度](https://leetcode.com/problems/online-stock-span/)

## 题目大意

设计一个算法，收集某股票的每日价格报价，并返回该股票当天的价格区间

股票价格在一天内的跨度是指从当天开始向前追溯，股票价格低于或等于当天价格的连续天数的最大值。

- 例如，如果过去四天的股票价格为，[7,2,1,2]今天股票价格为2，那么今天的跨度为，4因为从今天开始，股票价格连续几天都小于或2等于4
- 此外，如果过去四天的股票价格为，[7,34,1,2]而今天股票的价格为8，那么今天的跨度为，3因为从今天开始，股票价格连续几天都小于或8等于3
  
实现该类StockSpanner：
- StockSpanner()初始化类的对象
- int next(int price)返回给定当前价格为 的股票价格的跨度price

```js
Example 1:
Input
["StockSpanner", "next", "next", "next", "next", "next", "next", "next"]
[[], [100], [80], [60], [70], [60], [75], [85]]
Output
[null, 1, 1, 1, 2, 1, 4, 6]

Explanation
StockSpanner stockSpanner = new StockSpanner();
stockSpanner.next(100); // return 1
stockSpanner.next(80);  // return 1
stockSpanner.next(60);  // return 1
stockSpanner.next(70);  // return 2
stockSpanner.next(60);  // return 1
stockSpanner.next(75);  // return 4, because the last 4 prices (including today's price of 75) were less than or equal to today's price.
stockSpanner.next(85);  // return 6
```

限制:
- 1 <= price <= 10^5
- At most 104 calls will be made to next.

## 解题

这题最适合用：`单调递减栈 + span 压缩`

栈里存什么？
- 栈里不是只存价格，而是存：[price, span]
- 也就是：当前价格，以及这个价格已经代表了多少连续天数，例如：[75, 4]，表示价格 75 这一天，已经覆盖了前面连续 4 天

为什么可以压缩 span？
- 假设之前有一个栈顶：[75, 4]
- 现在来了价格：85
  - 因为：85 >= 75
  - 说明当前 85 不仅可以覆盖 75 这一天，也可以覆盖 75 已经代表的那一整段连续天数
- 所以可以直接：span += 4，而不是一天一天重新数
- 这就是这题的核心优化

为什么栈是单调递减的？
- 只要栈顶价格小于等于当前价格，就弹出。所以栈里剩下的价格一定是从栈底到栈顶递减趋势
- 例如：[100, 80]
  - 如果来了 75，不能弹：100, 80, 75
  - 如果来了 85，会弹出 75 和 80，最后只剩：100, 85
- 所以**栈维护的是一个“还没有被未来更大价格覆盖掉”的价格集合**

```js
var StockSpanner = function() {
    this.stack = [];
};

/** 
 * @param {number} price
 * @return {number}
 */
StockSpanner.prototype.next = function(price) {
    let span = 1;

    while (
        this.stack.length > 0 &&
        this.stack[this.stack.length - 1][0] <= price
    ) {
        span += this.stack.pop()[1];
    }

    this.stack.push([price, span]);
    return span;
};
```
```python
class StockSpanner:
    def __init__(self):
        self.stack = []  # Each element: [price, span]

    def next(self, price: int) -> int:
        span = 1

        while self.stack and self.stack[-1][0] <= price:
            span += self.stack.pop()[1]

        self.stack.append([price, span])
        return span
```
```java
import java.util.*;

class StockSpanner {
    private Deque<int[]> stack;

    public StockSpanner() {
        // Each element is int[]{price, span}
        stack = new ArrayDeque<>();
    }

    public int next(int price) {
        int span = 1;

        while (!stack.isEmpty() && stack.peekLast()[0] <= price) {
            span += stack.pollLast()[1];
        }

        stack.offerLast(new int[]{price, span});
        return span;
    }
}
```

- 时间复杂度：O(n)，设一共调用了 n 次 next()，某一次调用可能会弹出很多元素 
  - 比如价格持续上涨：10, 20, 30, 40, 50，当 50 来的时候，可能会弹出前面多个价格。所以单次最坏是：O(n)
  - 均摊时间复杂度：O(1)，虽然某一次可能弹很多，但每个价格：最多入栈一次，最多出栈一次，所以所有 n 次调用加起来，总操作次数是线性的：O(n)
  - 因此平均到每次调用：`O(1)` amortized
- 空间复杂度：`O(n)`
  - 如果价格一直下降：100, 90, 80, 70, 60，没有元素会被弹出，所有价格都会留在栈里
  - 所以空间复杂度是：`O(n)`
