# 3562.折扣价交易股票的最大利润

题目链接：[3562.折扣价交易股票的最大利润](https://leetcode.cn/problems/maximum-profit-from-trading-stocks-with-discounts/)

## 题目大意

给你一个整数 n，表示公司中员工的数量。每位员工都分配了一个从 1 到 n 的唯一 ID ，其中员工 1 是 CEO，是每一个员工的直接或间接上司。另给你两个下标从 1 开始的整数数组 present 和 future，两个数组的长度均为 n，具体定义如下：
- present[i] 表示第 i 位员工今天可以购买股票的 当前价格 
- future[i] 表示第 i 位员工明天可以卖出股票的 预期价格 
  
公司的层级关系由二维整数数组 hierarchy 表示，其中 hierarchy[i] = [ui, vi] 表示员工 ui 是员工 vi 的直属上司

此外，再给你一个整数 budget，表示可用于投资的总预算

公司有一项折扣政策：如果某位员工的直属上司购买了公司的股票，那么该员工可以以 半价 购买股票（即 floor(present[v] / 2)）

请返回在不超过给定预算的情况下可以获得的 最大利润 

注意：
- 每只股票最多只能购买一次
- 不能使用股票未来的收益来增加投资预算，购买只能依赖于 budget

限制:
- 1 <= n <= 160
- present.length, future.length == n
- 1 <= present[i], future[i] <= 50
- hierarchy.length == n - 1
- hierarchy[i] == [ui, vi]
- 1 <= ui, vi <= n
- ui != vi
- 1 <= budget <= 160
- There are no duplicate edges.
- Employee 1 is the direct or indirect boss of every employee.
- The input graph hierarchy is guaranteed to have no cycles.

## 解题

**Tree DP + 树形背包 **

这题的核心是：每个员工买不买会影响他的直属下属是否能打折，所以这是一个带「父子状态依赖」的树形 DP。同时又有预算限制，所以每棵子树内部还要做「背包合并」

**建图**

题目给的 hierarchy 是上下级关系：manager -> employee

因为员工编号是从 1 开始，所以代码里转成 0-indexed

children = [[] for _ in range(n)]

for manager, employee in hierarchy:
    manager -= 1
    employee -= 1
    children[manager].append(employee)

这样 children[x] 表示员工 x 的所有直属下属

**DP 状态定义**

核心函数：dfs(employee) 返回当前员工这棵子树的 DP 表：dp[parent_bought][money]

含义是：在 employee 这棵子树中，如果 employee 的父节点是否买入状态为 parent_bought，并且刚好花费 money 预算，最多能获得多少利润

其中：
- parent_bought = 0：当前员工的父节点没买，所以当前员工不能打折
- parent_bought = 1：当前员工的父节点买了，所以当前员工可以买半价

为什么需要这个状态？
- 因为当前员工的购买价格取决于父节点是否买了：cost = present[employee] // (parent_bought + 1)

也就是：
- parent_bought = 0 -> cost = present[employee]
- parent_bought = 1 -> cost = present[employee] // 2

**先合并所有孩子子树**

代码中：

child_profit = [
    [0] + [NEG_INF] * budget,
    [0] + [NEG_INF] * budget
]

child_profit[current_bought][money] 表示：只考虑当前员工所有孩子子树时，当前员工是否买入为 current_bought，花费 money 能获得的最大利润

这里的 current_bought 很关键：

| 当前员工是否买 | 对孩子的影响                     |
| ------- | -------------------------- |
| 当前员工不买  | 孩子不能打折，所以孩子用 `child_dp[0]` |
| 当前员工买   | 孩子可以打折，所以孩子用 `child_dp[1]` |

所以合并孩子时：
- 如果 current_bought = 0，说明当前员工没买，孩子不能享受折扣
- 如果 current_bought = 1，说明当前员工买了，孩子可以享受折扣

**树形背包合并**

对于每个孩子 child，先递归拿到：child_dp = dfs(child)

然后把这个孩子子树合并进当前的 child_profit。

核心转移：

merged[total_budget] = max(
    merged[total_budget],
    child_profit[current_bought][previous_budget] + child_best_profit
)

意思是：
- 当前已经合并过一些孩子，花了 previous_budget，利润是 child_profit[...]
- 现在给新孩子分配 child_budget，利润是 child_best_profit
- 那么总预算就是 total_budget = previous_budget + child_budget

这就是典型的「分组背包 / 树形背包」合并

**再决定当前员工买不买**

孩子都合并完之后，开始处理当前员工自己

subtree_dp[parent_bought] = child_profit[0].copy()

这表示：
- 情况 1：不买当前员工
  - 如果不买当前员工，那么孩子不能打折
  - 所以直接用：child_profit[0]
- 情况 2：买当前员工
  - 如果买当前员工：
    - cost = present[employee] // (parent_bought + 1)
    - profit = future[employee] - cost
  - 然后孩子可以打折，所以用：child_profit[1]

转移：

subtree_dp[parent_bought][total_budget] = max(
    subtree_dp[parent_bought][total_budget],
    child_profit[1][remaining_budget] + profit
)

含义是：
- 花 cost 买当前员工，剩下 remaining_budget 给孩子子树
- 因为当前员工买了，所以孩子能打折，用 child_profit[1]
- 总利润 = 当前员工利润 + 孩子子树最大利润

**为什么最后返回 max(root_dp[0])**

原因有两个：
- 第一，root 没有父节点，所以 root 不能因为父节点买入而打折。因此用：root_dp[0] 表示 root 的父节点没买
- 第二，预算是「最多 budget」，不是「必须刚好花完 budget」，所以要在所有花费 0 ~ budget 的情况中取最大值：max(root_dp[0])

如果只返回：root_dp[0][budget]，意思更像「刚好花完 budget」，不如 max(root_dp[0]) 稳

**为什么用 -inf**

NEG_INF = -inf

因为这个 DP 表示的是：刚好花费某个预算时的最大利润

一开始只有花费 0 是合法的：[0] + [-inf] * budget

其他预算还没有被构造出来，所以是不可达状态，不能默认成 0

否则会错误地认为：不花钱也能达到任意预算状态

**为什么可以跳过负利润状态**

因为我们可以选择什么都不买，利润至少是 0

所以负利润状态不会帮助我们得到最大答案。

这个剪枝可以减少很多无用转移。

**整体思路总结**

这题可以这样理解：
- 对每个员工，我们需要知道两种情况：父节点买了还是没买
- 因为这决定当前员工是否能打折
- 对于当前员工的所有孩子，我们用背包方式合并他们的子树结果
- 合并完孩子后，再决定当前员工自己买不买
- 如果不买，孩子不能打折；如果买，孩子可以打折
- 最后 root 没有父节点，所以用状态 0，并在不超过 budget 的所有花费里取最大利润

```js
/**
 * @param {number} n
 * @param {number[]} present
 * @param {number[]} future
 * @param {number[][]} hierarchy
 * @param {number} budget
 * @return {number}
 */
var maxProfit = function(n, present, future, hierarchy, budget) {
    const children = Array.from({ length: n }, () => []);

    for (const [manager, employee] of hierarchy) {
        children[manager - 1].push(employee - 1);
    }

    const NEG_INF = Number.NEGATIVE_INFINITY;

    function dfs(employee) {
        let childProfit = [
            Array(budget + 1).fill(NEG_INF),
            Array(budget + 1).fill(NEG_INF)
        ];

        childProfit[0][0] = 0;
        childProfit[1][0] = 0;

        for (const child of children[employee]) {
            const childDp = dfs(child);

            for (let currentBought = 0; currentBought < 2; currentBought++) {
                const merged = Array(budget + 1).fill(NEG_INF);
                merged[0] = 0;

                for (let childBudget = 0; childBudget <= budget; childBudget++) {
                    const childBestProfit = childDp[currentBought][childBudget];

                    if (childBestProfit < 0) continue;

                    for (let totalBudget = childBudget; totalBudget <= budget; totalBudget++) {
                        const previousBudget = totalBudget - childBudget;

                        if (childProfit[currentBought][previousBudget] === NEG_INF) {
                            continue;
                        }

                        merged[totalBudget] = Math.max(
                            merged[totalBudget],
                            childProfit[currentBought][previousBudget] + childBestProfit
                        );
                    }
                }

                childProfit[currentBought] = merged;
            }
        }

        const subtreeDp = [
            Array(budget + 1).fill(NEG_INF),
            Array(budget + 1).fill(NEG_INF)
        ];

        for (let parentBought = 0; parentBought < 2; parentBought++) {
            // Option 1: skip current employee
            subtreeDp[parentBought] = [...childProfit[0]];

            // Option 2: buy current employee
            const cost = Math.floor(present[employee] / (parentBought + 1));
            const profit = future[employee] - cost;

            for (let totalBudget = cost; totalBudget <= budget; totalBudget++) {
                const remainingBudget = totalBudget - cost;

                if (childProfit[1][remainingBudget] === NEG_INF) {
                    continue;
                }

                subtreeDp[parentBought][totalBudget] = Math.max(
                    subtreeDp[parentBought][totalBudget],
                    childProfit[1][remainingBudget] + profit
                );
            }
        }

        return subtreeDp;
    }

    const rootDp = dfs(0);
    return Math.max(...rootDp[0]);
};
```
```python
from typing import List
from math import inf

class Solution:
    def maxProfit(
        self,
        n: int,
        present: List[int],
        future: List[int],
        hierarchy: List[List[int]],
        budget: int
    ) -> int:
        children = [[] for _ in range(n)]

        for manager, employee in hierarchy:
            manager -= 1
            employee -= 1
            children[manager].append(employee)

        NEG_INF = -inf

        def dfs(employee: int) -> List[List[int]]:
            """
            Return dp for this employee's subtree.

            dp[parent_bought][money] =
                maximum profit we can get from this subtree
                using exactly 'money' budget,
                where parent_bought indicates whether this employee's direct manager bought stock.

            parent_bought = 0:
                this employee cannot get discount

            parent_bought = 1:
                this employee can get discount
            """

            # child_profit[parent_bought_by_children][money]
            #
            # This stores the best profit from all child subtrees only.
            #
            # If current employee is NOT bought:
            #     children do not get discount, so use child dp[0].
            #
            # If current employee IS bought:
            #     children get discount, so use child dp[1].
            child_profit = [
                [0] + [NEG_INF] * budget,
                [0] + [NEG_INF] * budget
            ]

            # Merge each child's subtree DP into child_profit.
            for child in children[employee]:
                child_dp = dfs(child)

                for current_bought in range(2):
                    merged = [0] + [NEG_INF] * budget

                    # child_budget: budget allocated to this child subtree
                    for child_budget, child_best_profit in enumerate(child_dp[current_bought]):
                        if child_best_profit < 0:
                            continue

                        # total_budget: total budget used after merging this child
                        for total_budget in range(child_budget, budget + 1):
                            previous_budget = total_budget - child_budget

                            if child_profit[current_bought][previous_budget] == NEG_INF:
                                continue

                            merged[total_budget] = max(
                                merged[total_budget],
                                child_profit[current_bought][previous_budget] + child_best_profit
                            )

                    child_profit[current_bought] = merged

            # subtree_dp[parent_bought][money]
            subtree_dp = [[NEG_INF] * (budget + 1) for _ in range(2)]

            for parent_bought in range(2):
                # Option 1: do not buy current employee.
                # Then children cannot get discount, so use child_profit[0].
                subtree_dp[parent_bought] = child_profit[0].copy()

                # Option 2: buy current employee.
                # Current employee's cost depends on whether parent bought.
                cost = present[employee] // (parent_bought + 1)
                profit = future[employee] - cost

                if cost <= budget:
                    for total_budget in range(cost, budget + 1):
                        remaining_budget = total_budget - cost

                        if child_profit[1][remaining_budget] == NEG_INF:
                            continue

                        subtree_dp[parent_bought][total_budget] = max(
                            subtree_dp[parent_bought][total_budget],
                            child_profit[1][remaining_budget] + profit
                        )

            return subtree_dp

        root_dp = dfs(0)

        # Root has no parent, so parent_bought = 0.
        # We can use at most budget, not necessarily exactly budget.
        return max(root_dp[0])
```

```java
import java.util.*;

class Solution {
    private int[] present;
    private int[] future;
    private List<Integer>[] children;
    private int budget;
    private static final int NEG_INF = Integer.MIN_VALUE / 4;

    public int maxProfit(
        int n,
        int[] present,
        int[] future,
        int[][] hierarchy,
        int budget
    ) {
        this.present = present;
        this.future = future;
        this.budget = budget;

        children = new ArrayList[n];
        for (int i = 0; i < n; i++) {
            children[i] = new ArrayList<>();
        }

        for (int[] edge : hierarchy) {
            int manager = edge[0] - 1;
            int employee = edge[1] - 1;
            children[manager].add(employee);
        }

        int[][] rootDp = dfs(0);

        int answer = 0;
        for (int money = 0; money <= budget; money++) {
            answer = Math.max(answer, rootDp[0][money]);
        }

        return answer;
    }

    private int[][] dfs(int employee) {
        int[][] childProfit = new int[2][budget + 1];

        Arrays.fill(childProfit[0], NEG_INF);
        Arrays.fill(childProfit[1], NEG_INF);
        childProfit[0][0] = 0;
        childProfit[1][0] = 0;

        for (int child : children[employee]) {
            int[][] childDp = dfs(child);

            for (int currentBought = 0; currentBought < 2; currentBought++) {
                int[] merged = new int[budget + 1];
                Arrays.fill(merged, NEG_INF);
                merged[0] = 0;

                for (int childBudget = 0; childBudget <= budget; childBudget++) {
                    int childBestProfit = childDp[currentBought][childBudget];

                    if (childBestProfit < 0) {
                        continue;
                    }

                    for (int totalBudget = childBudget; totalBudget <= budget; totalBudget++) {
                        int previousBudget = totalBudget - childBudget;

                        if (childProfit[currentBought][previousBudget] == NEG_INF) {
                            continue;
                        }

                        merged[totalBudget] = Math.max(
                            merged[totalBudget],
                            childProfit[currentBought][previousBudget] + childBestProfit
                        );
                    }
                }

                childProfit[currentBought] = merged;
            }
        }

        int[][] subtreeDp = new int[2][budget + 1];

        for (int parentBought = 0; parentBought < 2; parentBought++) {
            Arrays.fill(subtreeDp[parentBought], NEG_INF);

            // Option 1: skip current employee
            for (int money = 0; money <= budget; money++) {
                subtreeDp[parentBought][money] = childProfit[0][money];
            }

            // Option 2: buy current employee
            int cost = present[employee] / (parentBought + 1);
            int profit = future[employee] - cost;

            for (int totalBudget = cost; totalBudget <= budget; totalBudget++) {
                int remainingBudget = totalBudget - cost;

                if (childProfit[1][remainingBudget] == NEG_INF) {
                    continue;
                }

                subtreeDp[parentBought][totalBudget] = Math.max(
                    subtreeDp[parentBought][totalBudget],
                    childProfit[1][remainingBudget] + profit
                );
            }
        }

        return subtreeDp;
    }
}
```

设：n = 员工数量，B = budget
- 时间复杂度
  - 每个节点都要合并它的孩子
  - 每合并一个孩子，本质上是背包合并：枚举 child_budget/枚举 total_budget
  - 最坏是：O(B^2)
  - 树上一共有 n - 1 条边，所以整体最坏时间复杂度是：O(n * B^2)
- 空间复杂度
  - 每个 DFS 返回一个大小为：2 * (B + 1) 的 DP 表
  - 递归过程中会保存子树 DP 和当前合并数组
  - 最坏空间可以写成：O(n * B)
  - 如果只看单层合并数组，不考虑递归返回表的累计占用，可以说每个节点的 DP 是：O(B)
