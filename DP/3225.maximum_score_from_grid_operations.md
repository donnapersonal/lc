# 3225.网格图操作后的最大分数

题目链接：[3225.网格图操作后的最大分数](https://leetcode.cn/problems/maximum-score-from-grid-operations/)

## 题目大意

给你一个大小为 n x n 的二维矩阵 grid ，一开始所有格子都是白色的。一次操作中，你可以选择任意下标为 (i, j) 的格子，并将第 j 列中从最上面到第 i 行所有格子改成黑色

如果格子 (i, j) 为白色，且左边或者右边的格子至少一个格子为黑色，那么我们将 grid[i][j] 加到最后网格图的总分中去

请你返回执行任意次操作以后，最终网格图的 最大 总分数

![example](https://github.com/donnapersonal/picx-images-hosting/raw/master/image.mpw4hw9e.webp)

限制：
- 1 <= n == grid.length <= 100
- n == grid[i].length
- 0 <= grid[i][j] <= 10^9

## 解题

**题目核心理解**

每一列可以选择把上面若干个格子染黑

假设第 j 列染黑高度是 h，表示：

第 j 列从第 0 行到第 h-1 行都染黑

所以每一列的状态可以用一个高度表示：

h = 0, 1, 2, ..., n

其中：

h = 0 表示这一列全白
h = n 表示这一列全黑

得分来自相邻列之间的黑白边界差异。
所以我们只需要关心相邻列的染黑高度关系

**预处理：每列前缀和**

代码：col_sum = [list(accumulate(col, initial=0)) for col in zip(*grid)]

含义：col_sum[j][h] = 第 j 列前 h 个格子的和

例如：

col_sum[j][3] = grid[0][j] + grid[1][j] + grid[2][j]

这样可以 O(1) 算出某一列从高度 a 到 b-1 的格子和：

col_sum[j][b] - col_sum[j][a]

**DP 状态含义**

代码：f = [[[0, 0] for _ in range(n + 1)] for _ in range(n)]

可以理解为：f[j][pre][dec]

表示：

处理到第 j 列时，
右边相邻列的黑格高度是 pre，
在某种趋势状态 dec 下能获得的最大分数。

这里不用死记 dec 的复杂含义，面试里可以简单解释为：

dec 用来区分右边高度变化趋势，避免某些凹形情况重复计分。

更直观地说：
- pre 表示当前要考虑的右侧高度
- dec 表示右侧是否处于下降趋势


**状态转移的三种主要情况**

在第 j 列向第 j + 1 列转移

设：
- cur = 第 j 列的黑格高度
- pre = 第 j+1 列的黑格高度

有三种主要关系
- 情况 1：cur == pre
  - 两列黑格高度一样。
  - 没有新的白格被计入得分
  - 所以直接继承：f[j][pre][0]
- 情况 2：cur < pre
  - 右边列黑得更多
  - 此时第 j 列中：[cur, pre)
  - 这一段可以贡献得分
  - 贡献值是：col_sum[j][pre] - col_sum[j][cur]
- 情况 3：cur > pre
  - 左边列黑得更多
  - 此时第 j + 1 列中：[pre, cur)
  - 这一段可以贡献得分

贡献值是：col_sum[j + 1][cur] - col_sum[j + 1][pre]

**为什么需要前缀最大值优化？**

如果直接枚举：j, pre, cur

复杂度是：`O(n^3)`

因为：

for j in range(n):
    for pre in range(n + 1):
        for cur in range(n + 1):

这可能太慢

观察转移时，对于某个 pre，我们经常需要求：
- cur < pre 的最大值
- cur > pre 的最大值

这种结构可以用：
- 前缀最大值
- 后缀最大值

优化掉一层循环

所以最终复杂度从：O(n^3)

优化成：O(n^2)

**前缀最大值部分**

代码：

pre_max = f[j][0][1] - col_sum[j][0]
for pre in range(1, n + 1):
    f[j + 1][pre][0] = f[j + 1][pre][1] = max(
        f[j][pre][0],
        pre_max + col_sum[j][pre]
    )
    pre_max = max(pre_max, f[j][pre][1] - col_sum[j][pre])

这部分主要处理：cur < pre

因为如果 cur < pre，贡献是：col_sum[j][pre] - col_sum[j][cur]

可以变形为：col_sum[j][pre] + (f[j][cur][1] - col_sum[j][cur])

对于固定的 pre：col_sum[j][pre] 是固定的

所以只需要维护之前所有 cur < pre 中最大的：f[j][cur][1] - col_sum[j][cur]

这就是：pre_max

**后缀最大值部分**

代码：

suf_max = f[j][n][0] + col_sum[j + 1][n]
for pre in range(n - 1, 0, -1):
    f[j + 1][pre][0] = max(
        f[j + 1][pre][0],
        suf_max - col_sum[j + 1][pre]
    )
    suf_max = max(suf_max, f[j][pre][0] + col_sum[j + 1][pre])

这部分主要处理：cur > pre

如果 cur > pre，贡献是：col_sum[j + 1][cur] - col_sum[j + 1][pre]

可以变形为：(f[j][cur][0] + col_sum[j + 1][cur]) - col_sum[j + 1][pre]

对于固定的 pre：-col_sum[j + 1][pre] 是固定的

所以维护右侧所有 cur > pre 中最大的：f[j][cur][0] + col_sum[j + 1][cur]

这就是：suf_max

**为什么单独处理 pre = 0？**

代码：

f[j + 1][0][0] = suf_max
f[j + 1][0][1] = max(f[j][0][0], f[j][n][0])

pre = 0 表示第 j + 1 列全白

这个状态比较特殊，因为会涉及“凹形”情况和避免重复计分，所以单独处理

面试中可以简单说：pre = 0 是边界状态，不能完全套用普通转移，所以单独更新

**最终答案**

代码：return max(f[-1][i][0] for i in range(n + 1))

意思是：枚举最后一列可能的黑格高度 i，取最大分数

**为什么只取状态 0？**

因为最后不需要再考虑右侧下降趋势，最终合法答案落在 dec = 0 的状态里

```js
/**
 * @param {number[][]} grid
 * @return {number}
 */
var maximumScore = function(grid) {
    const n = grid.length;

    const colSum = Array.from({ length: n }, () => Array(n + 1).fill(0));

    for (let c = 0; c < n; c++) {
        for (let r = 0; r < n; r++) {
            colSum[c][r + 1] = colSum[c][r] + grid[r][c];
        }
    }

    const f = Array.from({ length: n }, () =>
        Array.from({ length: n + 1 }, () => [0, 0])
    );

    for (let j = 0; j < n - 1; j++) {
        let preMax = f[j][0][1] - colSum[j][0];

        for (let pre = 1; pre <= n; pre++) {
            const value = Math.max(
                f[j][pre][0],
                preMax + colSum[j][pre]
            );

            f[j + 1][pre][0] = value;
            f[j + 1][pre][1] = value;

            preMax = Math.max(
                preMax,
                f[j][pre][1] - colSum[j][pre]
            );
        }

        let sufMax = f[j][n][0] + colSum[j + 1][n];

        for (let pre = n - 1; pre > 0; pre--) {
            f[j + 1][pre][0] = Math.max(
                f[j + 1][pre][0],
                sufMax - colSum[j + 1][pre]
            );

            sufMax = Math.max(
                sufMax,
                f[j][pre][0] + colSum[j + 1][pre]
            );
        }

        f[j + 1][0][0] = sufMax;
        f[j + 1][0][1] = Math.max(f[j][0][0], f[j][n][0]);
    }

    let res = 0;
    for (let h = 0; h <= n; h++) {
        res = Math.max(res, f[n - 1][h][0]);
    }

    return res;
};
```
```python
class Solution:
    def maximumScore(self, grid: List[List[int]]) -> int:
        n = len(grid)
        col_sum = [list(accumulate(col, initial=0)) for col in zip(*grid)]

        f = [[[0, 0] for _ in range(n + 1)] for _ in range(n)]
        for j in range(n - 1):
            # 用前缀最大值优化
            pre_max = f[j][0][1] - col_sum[j][0]
            for pre in range(1, n + 1):
                f[j + 1][pre][0] = f[j + 1][pre][1] = max(
                    f[j][pre][0],
                    pre_max + col_sum[j][pre]
                )
                pre_max = max(pre_max, f[j][pre][1] - col_sum[j][pre])

            # 用后缀最大值优化
            suf_max = f[j][n][0] + col_sum[j + 1][n]
            for pre in range(n - 1, 0, -1):
                f[j + 1][pre][0] = max(
                    f[j + 1][pre][0],
                    suf_max - col_sum[j + 1][pre]
                )
                suf_max = max(suf_max, f[j][pre][0] + col_sum[j + 1][pre])

            # 单独计算 pre=0 的状态
            f[j + 1][0][0] = suf_max
            f[j + 1][0][1] = max(f[j][0][0], f[j][n][0])

        return max(f[-1][i][0] for i in range(n + 1))
```
```java
import java.util.*;

class Solution {
    public long maximumScore(int[][] grid) {
        int n = grid.length;

        long[][] colSum = new long[n][n + 1];

        for (int c = 0; c < n; c++) {
            for (int r = 0; r < n; r++) {
                colSum[c][r + 1] = colSum[c][r] + grid[r][c];
            }
        }

        long[][][] f = new long[n][n + 1][2];

        for (int j = 0; j < n - 1; j++) {
            long preMax = f[j][0][1] - colSum[j][0];

            for (int pre = 1; pre <= n; pre++) {
                long value = Math.max(
                    f[j][pre][0],
                    preMax + colSum[j][pre]
                );

                f[j + 1][pre][0] = value;
                f[j + 1][pre][1] = value;

                preMax = Math.max(
                    preMax,
                    f[j][pre][1] - colSum[j][pre]
                );
            }

            long sufMax = f[j][n][0] + colSum[j + 1][n];

            for (int pre = n - 1; pre > 0; pre--) {
                f[j + 1][pre][0] = Math.max(
                    f[j + 1][pre][0],
                    sufMax - colSum[j + 1][pre]
                );

                sufMax = Math.max(
                    sufMax,
                    f[j][pre][0] + colSum[j + 1][pre]
                );
            }

            f[j + 1][0][0] = sufMax;
            f[j + 1][0][1] = Math.max(f[j][0][0], f[j][n][0]);
        }

        long res = 0;
        for (int h = 0; h <= n; h++) {
            res = Math.max(res, f[n - 1][h][0]);
        }

        return res;
    }
}
```

- 时间复杂度：`O(n^2)`，其中 `n` 是 `grid` 的长度。这是本题的最优复杂度，因为遍历 grid 就需要 `O(n^2)` 的时间了。
- 空间复杂度：`O(n^2)`

注：空间复杂度可以进一步优化至 O(n)，需要用到滚动数组，并在 DP 的过程中计算列的前缀和
