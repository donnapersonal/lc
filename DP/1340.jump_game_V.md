# 1340.跳跃游戏 V

题目链接：[1340.跳跃游戏 V](https://leetcode.cn/problems/jump-game-v/)

## 题目大意

给你一个整数数组 arr 和一个整数 d 。每一步你可以从下标 i 跳到：
- i + x ，其中 i + x < arr.length 且 0 < x <= d 
- i - x ，其中 i - x >= 0 且 0 < x <= d 
  
除此以外，你从下标 i 跳到下标 j 需要满足：arr[i] > arr[j] 且 arr[i] > arr[k] ，其中下标 k 是所有 i 到 j 之间的数字（更正式的，min(i, j) < k < max(i, j)）

你可以选择数组的任意下标开始跳跃。请你返回你 最多 可以访问多少个下标

请注意，任何时刻你都不能跳到数组的外面

![example](https://github.com/donnapersonal/picx-images-hosting/raw/master/image.6f1bvoag3h.webp)

限制:
- 1 <= arr.length <= 1000
- 1 <= arr[i] <= 10^5
- 1 <= d <= arr.length

## 解题

写个暴力搜索，枚举所有情况。
- 从 i 往左跳：枚举 j=i−1,i−2,…,max(i−d,0)。根据题目要求，遍历过的元素都必须严格小于 arr[i]，如果 arr[j]≥arr[i] 就停止枚举。然后从 j 开始继续跳
- 从 i 往右跳：枚举 j=i+1,i+2,…,min(i+d,n−1)。根据题目要求，遍历过的元素都必须严格小于 arr[i]，如果 arr[j]≥arr[i] 就停止枚举。然后从 j 开始继续跳
  
考虑到整个递归过程中有大量重复递归调用（递归入参相同）。由于递归函数没有副作用，同样的入参无论计算多少次，算出来的结果都是一样的，因此可以用记忆化搜索来优化：
- 如果一个状态（递归入参）是第一次遇到，那么可以在返回前，把状态及其结果记到一个 memo 数组中
- 如果一个状态不是第一次遇到（memo 中保存的结果不等于 memo 的初始值），那么可以直接返回 memo 中保存的结果

```js
/**
 * @param {number[]} arr
 * @param {number} d
 * @return {number}
 */
var maxJumps = function(arr, d) {
    const n = arr.length;
    const memo = Array(n).fill(0);

    function dfs(index) {
        if (memo[index] !== 0) {
            return memo[index];
        }

        let best = 1;

        // Try jumping left.
        for (let next = index - 1; next >= Math.max(index - d, 0); next--) {
            if (arr[next] >= arr[index]) {
                break;
            }

            best = Math.max(best, dfs(next) + 1);
        }

        // Try jumping right.
        for (let next = index + 1; next <= Math.min(index + d, n - 1); next++) {
            if (arr[next] >= arr[index]) {
                break;
            }

            best = Math.max(best, dfs(next) + 1);
        }

        memo[index] = best;
        return best;
    }

    let answer = 1;

    for (let i = 0; i < n; i++) {
        answer = Math.max(answer, dfs(i));
    }

    return answer;
};
```
```python
from typing import List
from functools import cache

class Solution:
    def maxJumps(self, arr: List[int], d: int) -> int:
        n = len(arr)

        @cache
        def dfs(index: int) -> int:
            best = 1

            # Try jumping to the left.
            for next_index in range(index - 1, max(index - d - 1, -1), -1):
                if arr[next_index] >= arr[index]:
                    break

                best = max(best, dfs(next_index) + 1)

            # Try jumping to the right.
            for next_index in range(index + 1, min(index + d + 1, n)):
                if arr[next_index] >= arr[index]:
                    break

                best = max(best, dfs(next_index) + 1)

            return best

        return max(dfs(i) for i in range(n))
```
```java
import java.util.*;

class Solution {
    private int[] arr;
    private int d;
    private int n;
    private int[] memo;

    public int maxJumps(int[] arr, int d) {
        this.arr = arr;
        this.d = d;
        this.n = arr.length;
        this.memo = new int[n];

        int answer = 1;

        for (int i = 0; i < n; i++) {
            answer = Math.max(answer, dfs(i));
        }

        return answer;
    }

    private int dfs(int index) {
        if (memo[index] != 0) {
            return memo[index];
        }

        int best = 1;

        // Try jumping left.
        for (int next = index - 1; next >= Math.max(index - d, 0); next--) {
            if (arr[next] >= arr[index]) {
                break;
            }

            best = Math.max(best, dfs(next) + 1);
        }

        // Try jumping right.
        for (int next = index + 1; next <= Math.min(index + d, n - 1); next++) {
            if (arr[next] >= arr[index]) {
                break;
            }

            best = Math.max(best, dfs(next) + 1);
        }

        memo[index] = best;
        return best;
    }
}
```

- 时间复杂度：`O(nd)`，其中 `n` 是 `arr` 的长度。由于每个状态只会计算一次，记忆化搜索的时间复杂度 = 状态个数 × 单个状态的计算时间。本题状态个数等于 `O(n)`，单个状态的计算时间为 `O(d)`，所以总的时间复杂度为 `O(nd)`
- 空间复杂度：`O(n)`。保存多少状态，就需要多少空间
