# 241.为运算表达式设计优先级

题目链接：[241.为运算表达式设计优先级](https://leetcode.cn/problems/different-ways-to-add-parentheses/)

## 题目大意

给你一个由数字和运算符组成的字符串 expression，按不同优先级组合数字和运算符，计算并返回所有可能组合的结果。你可以 按任意顺序 返回答案

生成的测试用例满足其对应输出值符合 32 位整数范围，不同结果的数量不超过 10^4 

```js
Example 1:
Input: expression = "2-1-1"
Output: [0,2]
Explanation:
((2-1)-1) = 0 
(2-(1-1)) = 2

Example 2:
Input: expression = "2*3-4*5"
Output: [-34,-14,-10,-10,10]
Explanation:
(2*(3-(4*5))) = -34 
((2*3)-(4*5)) = -14 
((2*(3-4))*5) = -10 
(2*((3-4)*5)) = -10 
(((2*3)-4)*5) = 10
```

限制：
- 1 <= expression.length <= 20
- expression consists of digits and the operator '+', '-', and '*'.
- All the integer values in the input expression are in the range [0, 99].
- The integer values in the input expression do not have a leading '-' or '+' denoting the sign.

## 解题

首先对 expression 做一个预处理，把全部的操作数（包括数字和算符）都放到 ops 数组中，因为题目数据满足每一个数字都是 [0,99] 的范围中，且算符总共有 3 个，所以分别用 `−1`，`−2`，`−3` 来表示算符 `+，−，∗`

因为对于表达式中的某一个算符 op，将其左部可能的计算结果用 left 集合来表示，其右部可能的计算结果用 right 集合来表示。那么以该算符为该表达式的最后一步操作的情况的全部可能结果就是对应集合 left 和集合 right 中元素对应该算符操作的组合数

那么枚举表达式中的全部算符来作为 left 和 right 的分隔符来求得对应的集合，那么该表达式最终的可能结果就是这些集合的并集

为了避免相同区间的重复计算，用 `dp[l][r]={v0, v1,…}` 来表示对应表达式 `ops[l:r]` 在按不同优先级组合数字和运算符的操作下能产生的全部可能结果

这样就可以通过记忆化搜索这种「自顶向下」的方式来进行求解原始的表达式的全部可能计算结果。而上面讨论的条件是需要计算的表达式中存在算符的情况，所以还需要讨论搜索结束的条件：当表达式不存在任何算符时，即 l=r 时，对应的结果集合中就只有该一个数字

dp[l][r]={ops[l]},l=r&ops[l] ≥ 0

```js
/**
 * @param {string} expression
 * @return {number[]}
 */
const ADDITION = -1;
const SUBTRACTION = -2;
const MULTIPLICATION = -3;

var diffWaysToCompute = function(expression) {
    const ops = [];

    let i = 0;
    const n = expression.length;

    while (i < n) {
        if (expression[i] >= '0' && expression[i] <= '9') {
            let x = 0;

            while (i < n && expression[i] >= '0' && expression[i] <= '9') {
                x = x * 10 + Number(expression[i]);
                i++;
            }

            ops.push(x);
        } else {
            if (expression[i] === '+') {
                ops.push(ADDITION);
            } else if (expression[i] === '-') {
                ops.push(SUBTRACTION);
            } else {
                ops.push(MULTIPLICATION);
            }
            i++;
        }
    }

    const memo = new Map();

    function dfs(l, r) {
        const key = `${l},${r}`;

        if (memo.has(key)) {
            return memo.get(key);
        }

        if (l === r) {
            return [ops[l]];
        }

        const res = [];

        for (let i = l; i < r; i += 2) {
            const left = dfs(l, i);
            const right = dfs(i + 2, r);
            const op = ops[i + 1];

            for (const x of left) {
                for (const y of right) {
                    if (op === ADDITION) {
                        res.push(x + y);
                    } else if (op === SUBTRACTION) {
                        res.push(x - y);
                    } else {
                        res.push(x * y);
                    }
                }
            }
        }

        memo.set(key, res);
        return res;
    }

    return dfs(0, ops.length - 1);
};
```

```java
import java.util.*;

class Solution {
    private static final int ADDITION = -1;
    private static final int SUBTRACTION = -2;
    private static final int MULTIPLICATION = -3;

    private List<Integer> ops;
    private Map<String, List<Integer>> memo;

    public List<Integer> diffWaysToCompute(String expression) {
        ops = new ArrayList<>();
        memo = new HashMap<>();

        int i = 0;
        int n = expression.length();

        while (i < n) {
            char ch = expression.charAt(i);

            if (Character.isDigit(ch)) {
                int x = 0;

                while (i < n && Character.isDigit(expression.charAt(i))) {
                    x = x * 10 + (expression.charAt(i) - '0');
                    i++;
                }

                ops.add(x);
            } else {
                if (ch == '+') {
                    ops.add(ADDITION);
                } else if (ch == '-') {
                    ops.add(SUBTRACTION);
                } else {
                    ops.add(MULTIPLICATION);
                }
                i++;
            }
        }

        return dfs(0, ops.size() - 1);
    }

    private List<Integer> dfs(int l, int r) {
        String key = l + "," + r;

        if (memo.containsKey(key)) {
            return memo.get(key);
        }

        if (l == r) {
            List<Integer> base = new ArrayList<>();
            base.add(ops.get(l));
            memo.put(key, base);
            return base;
        }

        List<Integer> res = new ArrayList<>();

        for (int i = l; i < r; i += 2) {
            List<Integer> left = dfs(l, i);
            List<Integer> right = dfs(i + 2, r);

            int op = ops.get(i + 1);

            for (int x : left) {
                for (int y : right) {
                    if (op == ADDITION) {
                        res.add(x + y);
                    } else if (op == SUBTRACTION) {
                        res.add(x - y);
                    } else {
                        res.add(x * y);
                    }
                }
            }
        }

        memo.put(key, res);
        return res;
    }
}
```

- 时间复杂度：`O(2^n)`，其中 n 为 ops 的大小。可以分析得到 n 一定是奇数，令 n=2k+1，最后的答案集合大小等于 C_k，第 k 个卡特兰数（Catalan Number）；以及 ∑k<n/2 C_k（计算中间结果所涉及的复杂度）限制在 `O(2^n)`
- 空间复杂度：`O(2^n)`，其中 n 为 ops 的大小
