# 174.地下城游戏

题目链接：[174.地下城游戏](https://leetcode.cn/problems/dungeon-game/)

## 题目大意

恶魔们抓住了公主并将她关在了地下城 dungeon 的 右下角 。地下城是由 m x n 个房间组成的二维网格。我们英勇的骑士最初被安置在 左上角 的房间里，他必须穿过地下城并通过对抗恶魔来拯救公主

骑士的初始健康点数为一个正整数。如果他的健康点数在某一时刻降至 0 或以下，他会立即死亡

有些房间由恶魔守卫，因此骑士在进入这些房间时会失去健康点数（若房间里的值为负整数，则表示骑士将损失健康点数）；其他房间要么是空的（房间里的值为 0），要么包含增加骑士健康点数的魔法球（若房间里的值为正整数，则表示骑士将增加健康点数）

为了尽快解救公主，骑士决定每次只 向右 或 向下 移动一步

返回确保骑士能够拯救到公主所需的最低初始健康点数

注意：任何房间都可能对骑士的健康点数造成威胁，也可能增加骑士的健康点数，包括骑士进入的左上角房间以及公主被监禁的右下角房间

![example](https://github.com/donnapersonal/picx-images-hosting/raw/master/image.77e6xvj8vu.webp)

限制：
- m == dungeon.length, n == dungeon[i].length
- 1 <= m, n <= 200
- -1000 <= dungeon[i][j] <= 1000

## 解题

这题的核心不是“从左上走到右下时血量怎么变化”，而是：倒着想：为了能从当前位置安全走到终点，我进入这个格子之前至少需要多少血量？

**为什么不能从左上往右下贪心？**

因为当前看起来好的路径，不一定最终最优

例如：
- 路径 A：前面加很多血，但后面扣很多血
- 路径 B：前面扣一点血，但后面很安全

所以不能只看当前累计血量

这题更适合 从终点反推起点：我想知道进入某个格子前至少需要多少血，才能保证后面活着走到终点。

**DP 状态定义**

原本二维 DP 可以定义为：dp[i][j] = 进入 dungeon[i][j] 之前，至少需要多少血量

但是这里用了一维数组：dp[j]

在处理第 i 行时，表示：进入当前行第 j 列这个格子之前，至少需要多少血量

也就是等价于二维里的：dp[i][j]

**状态转移怎么来？***

在格子 (i, j)，骑士下一步只能去：

右边：(i, j + 1)
下面：(i + 1, j)

所以进入当前格子后，我们希望选择更容易活下去的方向

也就是：min(dp[j], dp[j + 1])

这里：

一维 DP	对应二维含义
dp[j]	下面格子 (i + 1, j) 所需血量
dp[j + 1]	右边格子 (i, j + 1) 所需血量

**为什么是这样？**

因为从右下往左上遍历：

for i in range(n - 1, -1, -1):
    for j in range(m - 1, -1, -1):

所以：

dp[j] 还没被当前行更新，仍然表示“下面”的状态
dp[j + 1] 已经在当前行更新过，表示“右边”的状态

**转移公式**

假设下一步最低需要：min(dp[j], dp[j + 1])

当前格子会影响血量：dungeon[i][j]

所以进入当前格子前至少需要：min(dp[j], dp[j + 1]) - dungeon[i][j]

**为什么要减？**

因为：

当前格子值	影响
-5	扣 5 血，所以进入前要多准备 5
+10	加 10 血，所以进入前可以少准备
0	不影响

但是血量最低必须是 1，所以：dp[j] = max(min(dp[j], dp[j + 1]) - dungeon[i][j], 1)

这就是你的核心代码

**为什么初始化是这样？**

这里用：BIG = 10**9

dp = [BIG] * (m + 1)
dp[m - 1] = 1

我们多开一格 m + 1，是为了模拟边界

初始：dp = [BIG, BIG, ..., BIG]

然后设置：dp[m - 1] = 1

意思是：`在公主房间的“出口方向”上，最低需要 1 点血量`

这样在计算右下角时，可以正确得到答案

**举个右下角例子**

假设最后一个格子是：dungeon[n-1][m-1] = -5

那么右下角计算时：dp[m - 1] = max(min(dp[m - 1], dp[m]) - dungeon[n-1][m-1], 1)

此时：

dp[m - 1] = 1
dp[m] = BIG

所以：min(1, BIG) = 1

然后：

max(1 - (-5), 1)
= max(6, 1)
= 6

意思是：进入最后一个格子前至少要有 6 点血，扣掉 5 后还剩 1，刚好活着

正确

如果最后一个格子是：dungeon[n-1][m-1] = 10

那么：

max(1 - 10, 1)
= max(-9, 1)
= 1

意思是：进入最后一个格子前只需要 1 点血，因为这个格子会回血

也正确

```js
/**
 * @param {number[][]} dungeon
 * @return {number}
 */
var calculateMinimumHP = function(dungeon) {
    const n = dungeon.length;
    const m = dungeon[0].length;

    const BIG = 1e9;
    const dp = new Array(m + 1).fill(BIG);

    dp[m - 1] = 1;

    for (let i = n - 1; i >= 0; i--) {
        for (let j = m - 1; j >= 0; j--) {
            const nextNeed = Math.min(dp[j], dp[j + 1]);
            dp[j] = Math.max(nextNeed - dungeon[i][j], 1);
        }
    }

    return dp[0];
};
```
```python
class Solution:
    def calculateMinimumHP(self, dungeon: List[List[int]]) -> int:
        n, m = len(dungeon), len(dungeon[0])
        BIG = 10**9
        dp = [[BIG] * (m + 1) for _ in range(n + 1)]
        dp[n][m - 1] = dp[n - 1][m] = 1
        for i in range(n - 1, -1, -1):
            for j in range(m - 1, -1, -1):
                minn = min(dp[i + 1][j], dp[i][j + 1])
                dp[i][j] = max(minn - dungeon[i][j], 1)

        return dp[0][0]

# 空间优化：rolling 数组
```

```java
class Solution {
    public int calculateMinimumHP(int[][] dungeon) {
        int n = dungeon.length;
        int m = dungeon[0].length;

        int BIG = 1_000_000_000;
        int[] dp = new int[m + 1];

        for (int j = 0; j <= m; j++) {
            dp[j] = BIG;
        }

        dp[m - 1] = 1;

        for (int i = n - 1; i >= 0; i--) {
            for (int j = m - 1; j >= 0; j--) {
                int nextNeed = Math.min(dp[j], dp[j + 1]);
                dp[j] = Math.max(nextNeed - dungeon[i][j], 1);
            }
        }

        return dp[0];
    }
}
```

- 时间复杂度：`O(N×M)`，其中 `N,M` 为给定矩阵的长宽
- 空间复杂度：`O(N×M)`，其中 `N,M` 为给定矩阵的长宽，注意这里可以利用滚动数组进行优化，优化后空间复杂度可以达到 `O(N)`
