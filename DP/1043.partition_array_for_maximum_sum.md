# 1043.分隔数组以得到最大和

题目链接：[1043.分隔数组以得到最大和](https://leetcode.cn/problems/partition-array-for-maximum-sum/)

## 题目大意

给你一个整数数组 arr，请你将该数组分隔为长度 最多 为 k 的一些（连续）子数组。分隔完成后，每个子数组的中的所有值都会变为该子数组中的最大值

返回将数组分隔变换后能够得到的元素最大和。本题所用到的测试用例会确保答案是一个 32 位整数

```js
Example 1:
Input: arr = [1,15,7,9,2,5,10], k = 3
Output: 84
Explanation: arr becomes [15,15,15,9,10,10,10]

Example 2:
Input: arr = [1,4,1,5,7,3,6,1,9,9,3], k = 4
Output: 83

Example 3:
Input: arr = [1], k = 1
Output: 1
```

限制：
- 1 <= arr.length <= 500
- 0 <= arr[i] <= 10^9
- 1 <= k <= arr.length

## 解题

**核心思路：`动态规划`**

这题最关键的是：`最后一段怎么切？`

假设现在考虑数组前 `i + 1` 个元素，也就是：`arr[0 : i + 1]`

最后一段可能是：

arr[i : i + 1]
arr[i - 1 : i + 1]
arr[i - 2 : i + 1]
...

但是最后一段长度不能超过 `k`

所以我们可以枚举最后一段的起点 `j`

最后一段就是：`arr[j : i + 1]`

前面的部分就是：`arr[0 : j]`

如果前面部分的最大答案已经知道，那么当前答案就可以算出来

**DP 状态定义**

定义：`f[i] = arr` 前 `i` 个元素可以得到的最大和

注意这里是 前 `i` 个元素，不是下标 `i`

所以：`f[0] = 0` 表示前 `0` 个元素的最大和是 `0`

如果我们正在处理下标 `i`，也就是要计算：`f[i + 1]`

**状态转移**

假设最后一段是：`arr[j : i + 1]`

它的长度是：`i - j + 1`

它的最大值是：`mx`

那么这最后一段变换后的贡献就是：`mx * (i - j + 1)`

前面 `arr[0 : j]` 的最大和是：`f[j]`

所以候选答案是：`f[j] + mx * (i - j + 1)`

因此转移公式是：`f[i + 1] = max(f[i + 1], f[j] + mx * (i - j + 1))`

**为什么倒着枚举 j？**

对于每个结尾位置 `i`，我们枚举最后一段的起点 `j`：`j = i, i - 1, i - 2, ...`

最多往前看 k 个元素。

也就是说：最后一段长度不能超过 `k`

所以 `j` 的范围是：`i, i - 1, ..., max(i - k + 1, 0)`

代码中写成：`for j in range(i, max(i - k, -1), -1):`

这个写法的意思是：从 i 往前枚举，但不能超过 k 个元素

同时我们一边枚举，一边更新当前最后一段的最大值：`mx = max(mx, arr[j])`

这样就不用每次重新扫描 `arr[j : i + 1]` 找最大值了

```js
/**
 * @param {number[]} arr
 * @param {number} k
 * @return {number}
 */
var maxSumAfterPartitioning = function(arr, k) {
    const n = arr.length;
    const f = new Array(n + 1).fill(0);

    for (let i = 0; i < n; i++) {
        let mx = 0;

        for (let j = i; j > Math.max(i - k, -1); j--) {
            mx = Math.max(mx, arr[j]);
            f[i + 1] = Math.max(
                f[i + 1],
                f[j] + (i - j + 1) * mx
            );
        }
    }

    return f[n];
};
```

```java
class Solution {
    public int maxSumAfterPartitioning(int[] arr, int k) {
        int n = arr.length;
        int[] f = new int[n + 1];

        for (int i = 0; i < n; i++) {
            int mx = 0;

            for (int j = i; j > Math.max(i - k, -1); j--) {
                mx = Math.max(mx, arr[j]);
                f[i + 1] = Math.max(
                    f[i + 1],
                    f[j] + (i - j + 1) * mx
                );
            }
        }

        return f[n];
    }
}
```

- 时间复杂度：`O(nk)`，其中 `n` 为 `arr` 的长度。动态规划的时间复杂度 = 状态个数 × 单个状态的计算时间。这里状态个数为 `O(n)`，单个状态的计算时间为 `O(k)`，因此时间复杂度为 `O(nk)`
- 空间复杂度：`O(n)`
