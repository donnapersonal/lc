# 3418.机器人可以获得的最大金币数

题目链接：[3418.机器人可以获得的最大金币数](https://leetcode.cn/problems/maximum-amount-of-money-robot-can-earn/)

## 题目大意

给你一个 m x n 的网格。一个机器人从网格的左上角 (0, 0) 出发，目标是到达网格的右下角 (m - 1, n - 1)。在任意时刻，机器人只能向右或向下移动

网格中的每个单元格包含一个值 coins[i][j]：

- 如果 coins[i][j] >= 0，机器人可以获得该单元格的金币
- 如果 coins[i][j] < 0，机器人会遇到一个强盗，强盗会抢走该单元格数值的 绝对值 的金币

机器人有一项特殊能力，可以在行程中 最多感化 2个单元格的强盗，从而防止这些单元格的金币被抢走

注意：机器人的总金币数可以是负数

返回机器人在路径上可以获得的 最大金币数 

```js
Example 1:
Input: coins = [[0,1,-1],[1,-2,3],[2,-3,4]]
Output: 8
Explanation:
An optimal path for maximum coins is:
Start at (0, 0) with 0 coins (total coins = 0).
Move to (0, 1), gaining 1 coin (total coins = 0 + 1 = 1).
Move to (1, 1), where there's a robber stealing 2 coins. The robot uses one neutralization here, avoiding the robbery (total coins = 1).
Move to (1, 2), gaining 3 coins (total coins = 1 + 3 = 4).
Move to (2, 2), gaining 4 coins (total coins = 4 + 4 = 8).

Example 2:
Input: coins = [[10,10,10],[10,10,10]]
Output: 40
Explanation:
An optimal path for maximum coins is:
Start at (0, 0) with 10 coins (total coins = 10).
Move to (0, 1), gaining 10 coins (total coins = 10 + 10 = 20).
Move to (0, 2), gaining another 10 coins (total coins = 20 + 10 = 30).
Move to (1, 2), gaining the final 10 coins (total coins = 30 + 10 = 40).
```

限制：
- m == coins.length, n == coins[i].length
- 1 <= m, n <= 500
- -1000 <= coins[i][j] <= 1000

## 解题

这题可以理解为：你从左上角走到右下角，每次只能：向右走、向下走

每个格子有一个金币值：

正数：得到金币
负数：遇到强盗，会损失金币
你最多可以使用 2 次特殊能力，把某个负数格子的损失抵消掉，相当于这个格子贡献 0

目标是：在最多抵消 2 个负数格子的情况下，拿到的最大金币数

**为什么要用 DP？**
- 因为每个格子只能从两个方向来：上方、左方

到达当前格子的最大收益，取决于：
- 到达上方格子的最大收益
- 到达左方格子的最大收益
- 当前格子的金币值
- 还能不能使用抵消机会

所以这是一个典型的 路径 DP + 状态 DP

**DP 状态定义**

你的代码使用了一维数组：f = [[-inf] * 3 for _ in range(n + 1)]

其中：
- f[j][0] 表示走到当前行第 j-1 列时，最多使用 0 次抵消机会的最大金额
- f[j][1] 表示走到当前行第 j-1 列时，最多使用 1 次抵消机会的最大金额
- f[j][2] 表示走到当前行第 j-1 列时，最多使用 2 次抵消机会的最大金额

注意这里用了 n + 1 的长度，所以真实列号 j 对应的是：f[j + 1]

这是为了做 padding，让边界更好处理。

**为什么初始化 f[1] = [0] * 3？**

代码：f[1] = [0] * 3

意思是：在起点之前，金额为 0

这个初始化相当于设置一个虚拟入口，让第一个格子 (0, 0) 可以正常通过转移算出来

如果没有这个 padding 初始化，起点、第一行、第一列都需要单独处理

这里用：f[1] = [0, 0, 0] 可以统一处理所有格子

**状态转移怎么理解？**

当前格子值是：x

当前位置可以从：左边：f[j] 上边：f[j + 1] 来

因为是一维 DP：

表达式	含义
- f[j]	当前行左边格子的状态
- f[j + 1]	上一行同一列格子的状态

**f[j + 1][0] 的转移**

f[j + 1][0] = max(f[j][0], f[j + 1][0]) + x

这里表示：
- 没有任何抵消机会
- 所以无论当前格子是正数还是负数，都必须加上 x

可以从左边来：f[j][0] + x

也可以从上边来：f[j + 1][0] + x

所以取最大：max(f[j][0], f[j + 1][0]) + x

**f[j + 1][1] 的转移**

f[j + 1][1] = max(
    f[j][1] + x,
    f[j + 1][1] + x,
    f[j][0],
    f[j + 1][0]
)

这里表示：最多可以使用 1 次抵消机会

有两类选
- 选择 1：不抵消当前格子
  - 那么当前格子的值 x 要正常加入
  - 从左边来：f[j][1] + x
  - 从上边来：f[j + 1][1] + x
- 选择 2：抵消当前格子
  - 如果我们选择抵消当前格子，那么当前格子的贡献就是 0
  - 但是会消耗一次抵消机会
  - 所以之前只能处于 0 次抵消状态
- 从左边来：f[j][0]
- 从上边来：f[j + 1][0]

注意这里没有 + x，因为当前格子被抵消了

所以最终取四种情况最大值

**f[j + 1][2] 的转移**

f[j + 1][2] = max(
    f[j][2] + x,
    f[j + 1][2] + x,
    f[j][1],
    f[j + 1][1]
)

这里表示：最多可以使用 2 次抵消机会

同样有两类选择
- 选择 1：不抵消当前格子
  - 从左边来：f[j][2] + x
  - 从上边来：f[j + 1][2] + x
- 选择 2：抵消当前格子
  - 消耗一次抵消机会，所以之前状态是 1 次抵消
  - 从左边来：f[j][1]
  - 从上边来：f[j + 1][1]
- 最终取最大

**为什么代码没有判断 x < 0？**

按题意，只有负数才值得抵消

但是代码里没有写：if x < 0:

为什么也对？

因为如果 x 是正数：
- 不抵消：previous + x
- 抵消：previous
- 显然：previous + x 更大，所以 max(...) 会自动选择“不抵消”

如果 x 是负数：
- 不抵消：previous + x
- 抵消：previous
- 这时抵消可能更好，max(...) 会自动选择更优的
- 所以不需要单独判断正负

**为什么更新顺序是 2 → 1 → 0？**

你的代码顺序是：

f[j + 1][2] = ...
f[j + 1][1] = ...
f[j + 1][0] = ...

这个顺序很重要

因为计算 f[j + 1][2] 时，需要用到旧的：f[j + 1][1]

计算 f[j + 1][1] 时，需要用到旧的：f[j + 1][0]

如果先更新 0，再更新 1，就会把“上方格子的旧状态”覆盖掉，导致错误

所以必须从大状态往小状态更新：2 -> 1 -> 0

```js
/**
 * @param {number[][]} coins
 * @return {number}
 */
var maximumAmount = function(coins) {
    const n = coins[0].length;
    const NEG_INF = Number.NEGATIVE_INFINITY;

    const f = Array.from({ length: n + 1 }, () => [NEG_INF, NEG_INF, NEG_INF]);

    // padding 初始化：相当于起点左边的虚拟位置，金额为 0
    f[1] = [0, 0, 0];

    for (const row of coins) {
        for (let j = 0; j < n; j++) {
            const x = row[j];

            f[j + 1][2] = Math.max(
                f[j][2] + x,
                f[j + 1][2] + x,
                f[j][1],
                f[j + 1][1]
            );

            f[j + 1][1] = Math.max(
                f[j][1] + x,
                f[j + 1][1] + x,
                f[j][0],
                f[j + 1][0]
            );

            f[j + 1][0] = Math.max(
                f[j][0],
                f[j + 1][0]
            ) + x;
        }
    }

    return f[n][2];
};
```

# 3D DP
class Solution:
    def maximumAmount(self, coins: List[List[int]]) -> int:
        m, n = len(coins), len(coins[0])
        f = [[[-inf] * 3 for _ in range(n + 1)] for _ in range(m + 1)]
        f[0][1] = [0] * 3
        for i, row in enumerate(coins):
            for j, x in enumerate(row):
                f[i + 1][j + 1][0] = max(f[i + 1][j][0], f[i][j + 1][0]) + x
                f[i + 1][j + 1][1] = max(f[i + 1][j][1] + x, f[i][j + 1][1] + x,
                                         f[i + 1][j][0], f[i][j + 1][0])
                f[i + 1][j + 1][2] = max(f[i + 1][j][2] + x, f[i][j + 1][2] + x,
                                         f[i + 1][j][1], f[i][j + 1][1])
        return f[m][n][2]

```java
class Solution {
    public int maximumAmount(int[][] coins) {
        int n = coins[0].length;
        int NEG_INF = Integer.MIN_VALUE / 2;

        int[][] f = new int[n + 1][3];

        for (int j = 0; j <= n; j++) {
            for (int k = 0; k < 3; k++) {
                f[j][k] = NEG_INF;
            }
        }

        // padding 初始化：相当于起点左边的虚拟位置，金额为 0
        f[1][0] = 0;
        f[1][1] = 0;
        f[1][2] = 0;

        for (int[] row : coins) {
            for (int j = 0; j < n; j++) {
                int x = row[j];

                f[j + 1][2] = Math.max(
                    Math.max(f[j][2] + x, f[j + 1][2] + x),
                    Math.max(f[j][1], f[j + 1][1])
                );

                f[j + 1][1] = Math.max(
                    Math.max(f[j][1] + x, f[j + 1][1] + x),
                    Math.max(f[j][0], f[j + 1][0])
                );

                f[j + 1][0] = Math.max(f[j][0], f[j + 1][0]) + x;
            }
        }

        return f[n][2];
    }
}
```

- 时间复杂度：`O(mn)`，其中 `m` 和 `n` 分别为 `coins` 的行数和列数
- 空间复杂度：`O(n)`
