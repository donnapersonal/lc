# 3600.升级后最大生成树稳定性

题目链接：[3600.升级后最大生成树稳定性](https://leetcode.cn/problems/maximize-spanning-tree-stability-with-upgrades/)

## 题目大意

给你一个整数 n，表示编号从 0 到 n - 1 的 n 个节点，以及一个 edges 列表，其中 edges[i] = [ui, vi, si, musti]：

- ui 和 vi 表示节点 ui 和 vi 之间的一条无向边
- si 是该边的强度
- musti 是一个整数（0 或 1），如果 musti == 1，则该边 必须 包含在生成树中，且不能升级 

你还有一个整数 k，表示你可以执行的最多 升级 次数。每次升级会使边的强度 翻倍，且每条可升级边（即 musti == 0）最多只能升级一次

一个生成树的 稳定性 定义为其中所有边的 最小 强度

返回任何有效生成树可能达到的 最大 稳定性，如果无法连接所有节点，返回 -1

注意： 图的一个 生成树（spanning tree）是该图中边的一个子集，它满足以下条件：

- 将所有节点连接在一起（即图是 连通的）
- 不 形成任何环
- 包含恰好 n - 1 条边，其中 n 是图中节点的数量

```js
Example 1:
Input: n = 3, edges = [[0,1,2,1],[1,2,3,0]], k = 1
Output: 2
Explanation:
Edge [0,1] with strength = 2 must be included in the spanning tree.
Edge [1,2] is optional and can be upgraded from 3 to 6 using one upgrade.
The resulting spanning tree includes these two edges with strengths 2 and 6.
The minimum strength in the spanning tree is 2, which is the maximum possible stability.

Example 2:
Input: n = 3, edges = [[0,1,4,0],[1,2,3,0],[0,2,1,0]], k = 2
Output: 6
Explanation:
Since all edges are optional and up to k = 2 upgrades are allowed.
Upgrade edges [0,1] from 4 to 8 and [1,2] from 3 to 6.
The resulting spanning tree includes these two edges with strengths 8 and 6.
The minimum strength in the tree is 6, which is the maximum possible stability.

Example 3:
Input: n = 3, edges = [[0,1,1,1],[1,2,1,1],[2,0,1,1]], k = 0
Output: -1
Explanation:
All edges are mandatory and form a cycle, which violates the spanning tree property of acyclicity. Thus, the answer is -1.
```

限制：
- 2 <= n <= 10^5
- 1 <= edges.length <= 10^5
- edges[i] = [ui, vi, si, musti]
- 0 <= ui, vi < n
- ui != vi
- 1 <= si <= 10^5
- musti is either 0 or 1.
- 0 <= k <= n
- There are no duplicate edges.

## 解题

### 思路 1: 二分查找 + 并查集

如：
- 当最小边权 `≥6` 时可以得到生成树 `T`，则当最小边权 `≥5`（或更小）时，我们可以选择同样的边，同样可以得到生成树 `T`
- 当最小边权 `≥7` 时无法得到生成树，则当最小边权 `≥8`（或更大）时，限制条件更加苛刻，更不可能得到生成树

据此，可以`二分猜答案`

**现在问题转化成一个判定性问题：**给定边权下界 `low`，能否得到一棵生成树，其边权都 `≥low`？
- 如果可以得到一棵生成树，说明答案 `≥low`，否则说明答案 `<low`

思路：
- 首先特判必选边成环的情况，此时无法得到生成树，返回 `−1`；或者，若原图不是连通的（连通块大于 `1`），也无法得到生成树，返回 `−1`
- 否则一定可以得到生成树，如 `low=min(si)` 的情况
- 然后二分猜答案，处理上文说的判定性问题
- 遍历 `edges`：
  - 对于必选边，以及边权 `≥low` 的边，无需操作，用并查集合并
  - 其余边，如果剩余升级次数大于 `0`，且边权乘 `2` 后 `≥low`，且边的两个端点不在同一个连通块中，那么用并查集合并，同时把剩余升级次数减一。注意这些需要操作的边没有先后顺序，只要最终可以得到一棵生成树即可
  - 遍历结束后，如果连通块个数等于 `1`，说明可以得到一棵生成树

> 这里采用开区间二分
> - 开区间左端点初始值：`min(si)`，无需操作，一定满足要求（注意已经提前特判返回 −1 的情况了）
> - 开区间右端点初始值：`max(si)⋅2+1`，加一后，一定无法满足要求
> 
> 对于开区间写法，简单来说 `check(mid) == true` 时更新的是谁，最后就返回谁。相比其他二分写法，开区间写法不需要思考加一减一等细节，更简单 -> 推荐使用开区间写二分

> 为什么二分结束后，答案 `res` 一定等于某个 `si` 或者 `2si`？
> - 反证法。假设 `res` 不等于任何 `si` 或 `2si`，这意味着生成树的最小边权大于 `res`，即 `≥res+1`，换句话说，`check(res+1)=true`
> - 但根据循环不变量，二分结束后 `check(res+1)=false`，矛盾
> - 故原命题成立

class UF {
    constructor(n) {
        this.parent = Array.from({ length: n }, (_, i) => i);
        this.count = n;
    }

    find(x) {
        if (this.parent[x] !== x) {
        this.parent[x] = this.find(this.parent[x]);
        }
        return this.parent[x];
    }

    merge(u, v) {
        const pu = this.find(u);
        const pv = this.find(v);
        if (pu === pv) return false;

        this.parent[pu] = pv;
        this.count--;
        return true;
    }
}

/**
 * @param {number} n
 * @param {number[][]} edges
 * @param {number} k
 * @return {number}
 */
var maxStability = function (n, edges, k) {
    if (n === 1) return 0;

    const mustUf = new UF(n);
    const allUf = new UF(n);

    let minS = Infinity;
    let maxS = 0;

    for (const [x, y, s, must] of edges) {
        if (must === 1 && !mustUf.merge(x, y)) {
        return -1;
        }

        allUf.merge(x, y);
        minS = Math.min(minS, s);
        maxS = Math.max(maxS, s);
    }

    if (allUf.count > 1) return -1;

    const check = (low) => {
        const uf = new UF(n);

        // 先加入必须边，以及本身就 >= low 的边
        for (const [x, y, s, must] of edges) {
        if (must === 1 && s < low) return false;

        if (must === 1 || s >= low) {
            uf.merge(x, y);
        }
        }

        let leftK = k;

        // 再尝试使用可升级边
        for (const [x, y, s, must] of edges) {
        if (leftK === 0 || uf.count === 1) break;

        if (must === 0 && s < low && low <= s * 2) {
            if (uf.merge(x, y)) {
            leftK--;
            }
        }
        }

        return uf.count === 1;
    };

    let left = minS;
    let right = maxS * 2 + 1; // 开区间上界

    while (left + 1 < right) {
        const mid = Math.floor((left + right) / 2);
        if (check(mid)) {
        left = mid;
        } else {
        right = mid;
        }
    }

    return left;
};

class UF:
    def __init__(self, n: int):
        self.meta = list(range(n))
        self.count = n
    
    def find(self, x: int) -> int:
        if self.meta[x] != x:
            self.meta[x] = self.find(self.meta[x])
        
        return self.meta[x]
    
    def merge(self, u: int, v: int) -> bool:
        x, y = self.find(u), self.find(v)
        if x == y:
            return False
        
        self.meta[x] = y
        self.count -= 1
        return True

class Solution:
    def maxStability(self, n: int, edges: List[List[int]], k: int) -> int:
        must_uf = UF(n)
        all_uf = UF(n)
        min_s, max_s = inf, 0
        for x, y, s, must in edges:
            if must and not must_uf.merge(x, y):
                return -1
            
            all_uf.merge(x, y)
            min_s = min(min_s, s)
            max_s = max(max_s, s)
        
        if all_uf.count > 1:
            return -1
        
        def check(low: int) -> bool:
            uf = UF(n)
            for x, y, s, must in edges:
                if must and s < low:
                    return False
                
                if must or s >= low:
                    uf.merge(x, y)
            
            left_k = k
            for x, y, s, must in edges:
                if left_k == 0 or uf.count == 1:
                    break
                
                if not must and s < low <= s * 2 and uf.merge(x, y):
                    left_k -= 1
            
            return uf.count == 1

        left, right = min_s, max_s * 2 + 1
        while left + 1 < right:
            mid = (left + right) // 2
            if check(mid):
                left = mid
            else:
                right = mid

        return left

```java
class UnionFind {
    private final int[] meta; // 代表元
    public int count; // 连通块个数

    UnionFind(int n) {
        // 一开始有 n 个集合 {0}, {1}, ..., {n-1}
        // 集合 i 的代表元是自己
        meta = new int[n];
        for (int i = 0; i < n; i++) {
            meta[i] = i;
        }
        count = n;
    }

    // 返回 x 所在集合的代表元
    // 同时做路径压缩，也就是把 x 所在集合中的所有元素的 meta 都改成代表元
    public int find(int x) {
        // 如果 meta[x] == x，则表示 x 是代表元
        if (meta[x] != x) {
            meta[x] = find(meta[x]); // meta 改成代表元
        }
        return meta[x];
    }

    // 把 from 所在集合合并到 to 所在集合中
    // 返回是否合并成功
    public boolean merge(int from, int to) {
        int x = find(from);
        int y = find(to);
        if (x == y) { // from 和 to 在同一个集合，不做合并
            return false;
        }
        meta[x] = y; // 合并集合。修改后就可以认为 from 和 to 在同一个集合了
        count--; // 成功合并，连通块个数减一
        return true;
    }
}

class Solution {
    public int maxStability(int n, int[][] edges, int k) {
        UnionFind mustUf = new UnionFind(n); // 必选边并查集
        UnionFind allUf = new UnionFind(n); // 全图并查集
        int minS = Integer.MAX_VALUE;
        int maxS = 0;
        for (int[] e : edges) {
            int x = e[0], y = e[1], s = e[2], must = e[3];
            if (must > 0 && !mustUf.merge(x, y)) { // 必选边成环
                return -1;
            }
            allUf.merge(x, y);
            minS = Math.min(minS, s);
            maxS = Math.max(maxS, s);
        }

        if (allUf.count > 1) { // 图不连通
            return -1;
        }

        int left = minS;
        int right = maxS * 2 + 1;
        while (left + 1 < right) {
            int mid = left + (right - left) / 2;
            if (check(mid, n, edges, k)) {
                left = mid;
            } else {
                right = mid;
            }
        }
        return left;
    }

    private boolean check(int low, int n, int[][] edges, int k) {
        UnionFind u = new UnionFind(n);
        for (int[] e : edges) {
            int x = e[0], y = e[1], s = e[2], must = e[3];
            if (must > 0 && s < low) { // 必选边的边权太小
                return false;
            }
            if (must > 0 || s >= low) {
                u.merge(x, y);
            }
        }

        for (int[] e : edges) {
            if (k == 0 || u.count == 1) {
                break;
            }
            int x = e[0], y = e[1], s = e[2], must = e[3];
            if (must == 0 && s < low && s * 2 >= low && u.merge(x, y)) {
                k--;
            }
        }
        return u.count == 1;
    }
}
```

- 时间复杂度：`O((n+mlogn)logU)`，其中 `m` 是 `edges` 的长度，`U=max(si)−min(si)`（常数系数可以忽略）
- 空间复杂度：`O(n)`
  
> 注：如果改成缩点建图 + DFS/BFS 判断是否连通，可以做到 `O((n+m)logU)`

### 思路 2: Kruskal 算法

在选出必选边后，根据 `Kruskal` 算法求最大生成树，把剩余的边按照边权（先不乘 2）从大到小合并。若不按 `Kruskal` 算法计算，得到的生成树的最小边权只会更小

设在生成树中的非必选边的边权为 `a`（降序），例如 `a=[9,8,7,6,5]`，由于每条边至多操作一次，所以选 `a` 的后 `k` 个数乘以 `2`

操作后，答案为如下三者的最小值：
- 必选边中的最小边权
- 非必选边中的最小边权乘以 `2`。前提条件是 `k>0`，但实际无需判断，因为 `k=0` 的时候第三种情况更小
- 非必选边中的第 `k+1` 小边权
