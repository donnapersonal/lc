# 843.猜猜这个单词

题目链接：[843.猜猜这个单词](https://leetcode.cn/problems/guess-the-word/)

## 题目大意

给你一个由 不同 字符串组成的单词列表 words ，其中 words[i] 长度均为 6 。words 中的一个单词将被选作秘密单词 secret 

另给你一个辅助对象 Master ，你可以调用 Master.guess(word) 来猜单词，其中参数 word 长度为 6 且必须是 words 中的字符串

Master.guess(word) 将会返回如下结果：
- 如果 word 不是 words 中的字符串，返回 -1 ，或者
- 一个整数，表示你所猜测的单词 word 与 秘密单词 secret 的准确匹配（值和位置同时匹配）的数目
  
每组测试用例都会包含一个参数 allowedGuesses ，其中 allowedGuesses 是你可以调用 Master.guess(word) 的最大次数

对于每组测试用例，在不超过允许猜测的次数的前提下，你应该调用 Master.guess 来猜出秘密单词。最终，你将会得到以下结果：
- 如果你调用 Master.guess 的次数大于 allowedGuesses 所限定的次数或者你没有用 Master.guess 猜到秘密单词，则得到 "Either you took too many guesses, or you did not find the secret word." 
- 如果你调用 Master.guess 猜到秘密单词，且调用 Master.guess 的次数小于或等于 allowedGuesses ，则得到 "You guessed the secret word correctly." 
  
生成的测试用例保证你可以利用某种合理的策略（而不是暴力）猜到秘密单词
```js
Example 1:
Input: secret = "acckzz", words = ["acckzz","ccbazz","eiowzz","abcczz"], allowedGuesses = 10
Output: You guessed the secret word correctly.
Explanation:
master.guess("aaaaaa") returns -1, because "aaaaaa" is not in words.
master.guess("acckzz") returns 6, because "acckzz" is secret and has all 6 matches.
master.guess("ccbazz") returns 3, because "ccbazz" has 3 matches.
master.guess("eiowzz") returns 2, because "eiowzz" has 2 matches.
master.guess("abcczz") returns 4, because "abcczz" has 4 matches.
We made 5 calls to master.guess, and one of them was the secret, so we pass the test case.

Example 2:
Input: secret = "hamada", words = ["hamada","khaled"], allowedGuesses = 10
Output: You guessed the secret word correctly.
Explanation: Since there are two words, you can guess both.
```

限制：
- 1 <= words.length <= 100
- words[i].length == 6
- words[i] consist of lowercase English letters.
- All the strings of words are unique.
- secret exists in words.
- 10 <= allowedGuesses <= 30

## 解题

这题是一个 `交互式猜词题`

我们不知道 secret，只能通过：`master.guess(word)` 得到当前猜测单词和 secret 有多少个位置字符相同

**核心思路**

维护一个候选集合 candidates，表示当前仍然可能是 secret 的单词

每次猜一个单词后，假设返回值是 matches，那么真正的 secret 必须满足：`match_count(secret, guess) == matches`

所以可以过滤候选集：

candidates = [
    word for word in candidates
    if match_count(word, guess) == matches
]

这样每猜一次，就能排除一部分不可能的单词

**关键问题：每次猜哪个词？**

不能随机猜，随机不稳定

这里使用 `minimax` 策略：对于每个候选单词 guess，我们模拟它会把当前候选集分成多少组：

match = 0 的一组
match = 1 的一组
match = 2 的一组
...
match = word_len 的一组

如果猜了这个词，API 返回哪个 match，我们就会保留对应那一组

所以每个 guess 的最坏情况就是：`max(groups)`，也就是最大那组的大小

我们选择：`最大组最小的 guess`

这样可以尽量保证无论 API 返回什么，剩下的候选词都尽可能少

代码逻辑：

def match_count(a, b):
    return sum(x == y for x, y in zip(a, b))

计算两个单词相同位置字符的数量

每一轮：

best_word = candidates[0]
best_worst_group = float("inf")

初始化当前最优猜测

然后枚举每个候选词作为 guess：

for guess in candidates:
    groups = [0] * (word_len + 1)

groups[i] 表示：如果 API 返回 i，还会剩下多少候选词

统计分组：

for word in candidates:
    if word != guess:
        groups[match_count(guess, word)] += 1

找到这个 guess 的最坏情况：worst_group = max(groups)

如果它比之前更好，就更新：

if worst_group < best_worst_group:
    best_worst_group = worst_group
    best_word = guess

然后真正调用 API：

matches = master.guess(best_word)

如果全部匹配：

if matches == word_len:
    return

说明已经猜中

否则过滤候选集：

candidates = [
    word for word in candidates
    if match_count(word, best_word) == matches
]

**为什么正确？**

每次 master.guess(best_word) 返回的 matches 是 secret 和 best_word 的真实匹配数

所以任何和 best_word 的匹配数不等于 matches 的单词，都不可能是 secret

因此过滤候选集是正确的

而 `minimax` 选词是为了让候选集在最坏情况下也尽量缩小

```js
/**
 * // This is the master's API interface.
 * // You should not implement it, or speculate about its implementation
 * function Master() {
 *
 *     @param {string} word
 *     @return {integer}
 *     this.guess = function(word) {
 *         ...
 *     };
 * };
 */
/**
 * @param {string[]} words
 * @param {Master} master
 * @return {void}
 */
var findSecretWord = function(words, master) {
    const matchCount = (a, b) => {
        let count = 0;

        for (let i = 0; i < a.length; i++) {
            if (a[i] === b[i]) {
                count++;
            }
        }

        return count;
    };

    let candidates = [...words];
    const wordLen = words[0].length;

    for (let round = 0; round < 30; round++) {
        let bestWord = candidates[0];
        let bestWorstGroup = Infinity;

        for (const guess of candidates) {
            const groups = new Array(wordLen + 1).fill(0);

            for (const word of candidates) {
                if (word !== guess) {
                    groups[matchCount(guess, word)]++;
                }
            }

            const worstGroup = Math.max(...groups);

            if (worstGroup < bestWorstGroup) {
                bestWorstGroup = worstGroup;
                bestWord = guess;
            }
        }

        const matches = master.guess(bestWord);

        if (matches === wordLen) {
            return;
        }

        candidates = candidates.filter(
            word => matchCount(word, bestWord) === matches
        );
    }
};
```

```java
/**
 * // This is the Master's API interface.
 * // You should not implement it, or speculate about its implementation
 * interface Master {
 *     public int guess(String word) {}
 * }
 */
import java.util.*;

class Solution {
    public void findSecretWord(String[] words, Master master) {
        List<String> candidates = new ArrayList<>(Arrays.asList(words));
        int wordLen = words[0].length();

        for (int round = 0; round < 30; round++) {
            String bestWord = candidates.get(0);
            int bestWorstGroup = Integer.MAX_VALUE;

            for (String guess : candidates) {
                int[] groups = new int[wordLen + 1];

                for (String word : candidates) {
                    if (!word.equals(guess)) {
                        groups[matchCount(guess, word)]++;
                    }
                }

                int worstGroup = 0;
                for (int count : groups) {
                    worstGroup = Math.max(worstGroup, count);
                }

                if (worstGroup < bestWorstGroup) {
                    bestWorstGroup = worstGroup;
                    bestWord = guess;
                }
            }

            int matches = master.guess(bestWord);

            if (matches == wordLen) {
                return;
            }

            List<String> nextCandidates = new ArrayList<>();

            for (String word : candidates) {
                if (matchCount(word, bestWord) == matches) {
                    nextCandidates.add(word);
                }
            }

            candidates = nextCandidates;
        }
    }

    private int matchCount(String a, String b) {
        int count = 0;

        for (int i = 0; i < a.length(); i++) {
            if (a.charAt(i) == b.charAt(i)) {
                count++;
            }
        }

        return count;
    }
}
```

设：

n = words 的数量
L = 单词长度
G = 最大猜测次数

- 时间复杂度：
  - 每一轮中：枚举每个 guess：O(n)，每个 guess 和所有候选词比较：O(n * L)
  - 所以每轮：`O(n^2 * L)`
  - 最多猜 G 轮：`O(G * n^2 * L)`
  - 题目中通常：L = 6，G = 10 或 30
  - 所以可以近似认为：`O(n^2)`
- 空间复杂度：`O(n + L)`，主要是候选集和分组数组
