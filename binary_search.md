一个很重要的思维模板：

以后你在面试里看到下面这种题型，都可以优先想“排序”：

模板 1：统计多少元素 >= target

比如这题：

对每个 spell，统计多少 potions >= needed
模板 2：找第一个满足条件的位置

排序后常变成：

第一个 >= target
第一个 > target
最后一个 < target
最后一个 <= target
模板 3：条件关于数值大小具有单调性

比如：

数越大越容易满足
数越小越容易满足

只要有这种单调关系，排序就很可能值得考虑。


一个很关键的思维判断标准

你以后可以这样区分：

优先想 binary search 的情况

当题目像是在问：

对每个 query
在有序数组中找边界

像这题：

每个 spell 都是一个 query
potions 是被查询数组

这时 binary search 很自然。

优先想 two pointers 的情况

当题目满足：

两边都能排序
指针可以单调移动
希望把多个 query 合并成一次扫描

这题也能被改造成这个形式，但不是第一眼最自然的形式。
