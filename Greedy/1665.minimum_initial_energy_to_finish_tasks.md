# 1665.完成所有任务的最少初始能量

题目链接：[1665.完成所有任务的最少初始能量](https://leetcode.cn/problems/minimum-initial-energy-to-finish-tasks/)

## 题目大意

给你一个任务数组 tasks ，其中 tasks[i] = [actuali, minimumi] ：
- actuali 是完成第 i 个任务 需要耗费 的实际能量
- minimumi 是开始第 i 个任务前需要达到的最低能量
  
比方说，如果任务为 [10, 12] 且你当前的能量为 11 ，那么你不能开始这个任务。如果你当前的能量为 13 ，你可以完成这个任务，且完成它后剩余能量为 3 

你可以按照 任意顺序 完成任务

请你返回完成所有任务的 最少 初始能量

```js
Example 1:
Input: tasks = [[1,2],[2,4],[4,8]]
Output: 8
Explanation:
Starting with 8 energy, we finish the tasks in the following order:
    - 3rd task. Now energy = 8 - 4 = 4.
    - 2nd task. Now energy = 4 - 2 = 2.
    - 1st task. Now energy = 2 - 1 = 1.
Notice that even though we have leftover energy, starting with 7 energy does not work because we cannot do the 3rd task.

Example 2:
Input: tasks = [[1,3],[2,4],[10,11],[10,12],[8,9]]
Output: 32
Explanation:
Starting with 32 energy, we finish the tasks in the following order:
    - 1st task. Now energy = 32 - 1 = 31.
    - 2nd task. Now energy = 31 - 2 = 29.
    - 3rd task. Now energy = 29 - 10 = 19.
    - 4th task. Now energy = 19 - 10 = 9.
    - 5th task. Now energy = 9 - 8 = 1.

Example 3:
Input: tasks = [[1,7],[2,8],[3,9],[4,10],[5,11],[6,12]]
Output: 27
Explanation:
Starting with 27 energy, we finish the tasks in the following order:
    - 5th task. Now energy = 27 - 5 = 22.
    - 2nd task. Now energy = 22 - 2 = 20.
    - 3rd task. Now energy = 20 - 3 = 17.
    - 1st task. Now energy = 17 - 1 = 16.
    - 4th task. Now energy = 16 - 4 = 12.
    - 6th task. Now energy = 12 - 6 = 6.
```

限制：
- 1 <= tasks.length <= 10^5
- 1 <= actual​i <= minimumi <= 10^4

## 解题

核心思路：`贪心排序`

我们希望优先做：门槛高，但实际消耗相对少的任务

也就是让：`minimum - actual` 越大，越靠前

代码里等价写成：`tasks.sort(key=lambda t: t[0] - t[1])`

因为：`actual - minimum 越小 = minimum - actual 越大`

**为什么这样排序？**

如果一个任务：minimum 很高，但 actual 不大，说明它对“开始前能量”的要求很高，但做完后消耗不算特别多 -》 这种任务如果放到后面，前面已经消耗了很多能量，可能就需要更高的初始能量 -》所以应该尽早做

变量含义
- s：当前已经累计消耗的能量
- res：到目前为止，所需的最小初始能量下界

对于当前任务：`[actual, minimum]`

如果已经消耗了 s，那么开始这个任务前，初始能量必须满足：`初始能量 - s >= minimum`

也就是：`初始能量 >= s + minimum`

所以更新：`res = max(res, s + minimum)`

做完任务后，累计消耗增加：`s += actual`

```js
/**
 * @param {number[][]} tasks
 * @return {number}
 */
var minimumEffort = function(tasks) {
    tasks.sort((a, b) => (a[0] - a[1]) - (b[0] - b[1]));

    let res = 0;
    let spent = 0;

    for (const [actual, minimum] of tasks) {
        res = Math.max(res, spent + minimum);
        spent += actual;
    }

    return res;
};
```
```python
class Solution:
    def minimumEffort(self, tasks: List[List[int]]) -> int:
        tasks.sort(key=lambda t: t[0] - t[1])  # actual - minimum 从小到大

        res = 0
        s = 0  # 已经累计消耗的能量

        for actual, minimum in tasks:
            res = max(res, s + minimum)
            s += actual

        return res
```
```java
import java.util.*;

class Solution {
    public int minimumEffort(int[][] tasks) {
        Arrays.sort(tasks, (a, b) -> {
            return (a[0] - a[1]) - (b[0] - b[1]);
        });

        int res = 0;
        int spent = 0;

        for (int[] task : tasks) {
            int actual = task[0];
            int minimum = task[1];

            res = Math.max(res, spent + minimum);
            spent += actual;
        }

        return res;
    }
}
```

- 时间复杂度：`O(nlogn)`，其中 `n` 是 `tasks` 的长度，瓶颈在排序上
- 空间复杂度：`O(1)`，不计入排序的栈开销
