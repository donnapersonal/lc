# 945.使数组唯一的最小增量

题目链接：[945.使数组唯一的最小增量](https://leetcode.cn/problems/minimum-increment-to-make-array-unique/)

## 题目大意

给你一个整数数组 nums 。每次 move 操作将会选择任意一个满足 0 <= i < nums.length 的下标 i，并将 nums[i] 递增 1

返回使 nums 中的每个值都变成唯一的所需要的最少操作次数

生成的测试用例保证答案在 32 位整数范围内

```js
Example 1:
Input: nums = [1,2,2]
Output: 1
Explanation: After 1 move, the array could be [1, 2, 3].

Example 2:
Input: nums = [3,2,1,2,1,7]
Output: 6
Explanation: After 6 moves, the array could be [3, 4, 1, 2, 5, 7].
It can be shown that it is impossible for the array to have all unique values with 5 or less moves.
```

限制：
- 1 <= nums.length <= 10^5
- 0 <= nums[i] <= 10^5

## 解题

这道题的意思是：给你一个数组，每次操作可以把某个数加 1。问最少需要多少次操作，才能让数组中所有数字都变成唯一的

例如：nums = [3, 2, 1, 2, 1, 7]，排序后：[1, 1, 2, 2, 3, 7]

我们希望每个数都唯一，并且操作次数最少

核心思路：`排序后从左到右贪心调整`

先排序：[1, 1, 2, 2, 3, 7]

排序后，如果想让所有数唯一，那么每个数都必须满足：`nums[i] > nums[i - 1]`，也就是说，当前数字必须严格大于前一个数字

如果当前数字已经比前一个数字大：`nums[i] > nums[i - 1]`，那就不用动

如果当前数字小于等于前一个数字：`nums[i] <= nums[i - 1]`，说明它和前面的数字冲突了，或者太小了

为了操作次数最少，应该把它变成：`nums[i - 1] + 1`
- 因为它必须比前一个数大，而最小能变成的合法值就是：`前一个数 + 1`
- 如果变得更大，只会多花操作次数，没有必要

为什么这个贪心是对的？
- 排序后，我们从左到右处理
- 对于当前位置 `i` 来说，前面的数字已经被调整好了，并且都是唯一递增的
- 现在 `nums[i]` 只有两种情况：
  - 情况 1：当前数已经合法 `nums[i] > nums[i - 1]`，不用操作
  - 情况 2：当前数不合法 `nums[i] <= nums[i - 1]`，它必须变得比前一个数大
- 最小合法值是：`nums[i - 1] + 1`，所以我们直接改成这个值

这就是局部最优：`当前数字只增加到刚好合法`

同时也是全局最优，因为多增加任何一步都会让总操作次数变多

```js
/**
 * @param {number[]} nums
 * @return {number}
 */
var minIncrementForUnique = function(nums) {
    nums.sort((a, b) => a - b);

    let moves = 0;

    for (let i = 1; i < nums.length; i++) {
        if (nums[i] <= nums[i - 1]) {
            const prev = nums[i];
            nums[i] = nums[i - 1] + 1;
            moves += nums[i] - prev;
        }
    }

    return moves;
};
```

```java
import java.util.Arrays;

class Solution {
    public int minIncrementForUnique(int[] nums) {
        Arrays.sort(nums);

        int moves = 0;

        for (int i = 1; i < nums.length; i++) {
            if (nums[i] <= nums[i - 1]) {
                int prev = nums[i];
                nums[i] = nums[i - 1] + 1;
                moves += nums[i] - prev;
            }
        }

        return moves;
    }
}
```

- 时间复杂度: `O(n log n)`，设数组长度为 `n`
  - 排序需要：`O(n log n)`
  - 后面遍历一遍数组需要：`O(n)`
  - 所以总时间复杂度是：`O(n log n)`
- 空间复杂度：`O(1)`
  - 如果不考虑排序内部使用的空间，这个算法只用了几个变量：`moves`, `i`, `prev`
  - 所以额外空间是：`O(1)`
  - 如果严格考虑不同语言排序的内部实现，可能会有额外排序空间，但面试中通常说：`O(1) extra space, ignoring sorting implementation details`
