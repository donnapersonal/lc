# 853.车队

题目链接：[853.车队](https://leetcode.com/problems/car-fleet/)

## 题目大意

n 距离起点 0 英里处有若干辆汽车，它们正在行驶以到达该英里处target

给定两个整数数组 position和speed，它们的长度均为n，其中position[i]是汽车的起始里程，是汽车的速度（英里/小时）。i^th speed[i] i^th

汽车不能超车，但可以追上速度较慢的汽车，然后以该车的速度与其并排行驶

车队是指一辆或多辆并排行驶的汽车。车队的速度是指车队中所有汽车的最低速度

如果一辆车在某英里处追上了一支车队target，它仍然会被视为车队的一部分

返回到达目的地的车队数量

```js
Example 1:
Input: target = 12, position = [10,8,0,5,3], speed = [2,4,1,1,3]
Output: 3
Explanation:
The cars starting at 10 (speed 2) and 8 (speed 4) become a fleet, meeting each other at 12. The fleet forms at target.
The car starting at 0 (speed 1) does not catch up to any other car, so it is a fleet by itself.
The cars starting at 5 (speed 1) and 3 (speed 3) become a fleet, meeting each other at 6. The fleet moves at speed 1 until it reaches target.

Example 2:
Input: target = 10, position = [3], speed = [3]
Output: 1
Explanation:
There is only one car, hence there is only one fleet.

Example 3:
Input: target = 100, position = [0,2,4], speed = [4,2,1]
Output: 1
Explanation:
The cars starting at 0 (speed 4) and 2 (speed 2) become a fleet, meeting each other at 4. The car starting at 4 (speed 1) travels to 5.
Then, the fleet at 4 (speed 2) and the car at position 5 (speed 1) become one fleet, meeting each other at 6. The fleet moves at speed 1 until it reaches target.
```

限制:
- n == position.length == speed.length
- 1 <= n <= 10^5
- 0 < target <= 10^6
- 0 <= position[i] < target
- All the values of position are unique.
- 0 < speed[i] <= 10^6

## 解题

题目给你：
- target = 终点位置
- position[i] = 第 i 辆车当前位置
- speed[i] = 第 i 辆车速度

每辆车都往 target 方向开

规则是：
- 后面的车如果追上前面的车，就会和前面的车组成一个车队，并且之后以较慢车的速度一起走
- 车不能超过前车

要求返回最后会有多少个车队到达终点


核心思路
- 这题核心是：按照 position 排序，然后从最靠近 target 的车开始往后扫描
- 为什么？因为后面的车只能追前面的车，不能超过前面的车
- 所以一辆车能不能形成新车队，取决于它能不能追上它前面的车队

判断是否形成新车队
- 我们维护一个变量：current_fleet_time 表示当前前方车队到达终点需要的时间
- 从右往左看每辆车：
  - 情况 1：当前车到终点时间 time <= current_fleet_time
    - 说明当前车比前方车队更快，或者一样快
    - 它可以在终点前追上前方车队
    - 所以：不会形成新车队
  - 情况 2：当前车到终点时间 time > current_fleet_time
    - 说明当前车比前方车队更慢
    - 它追不上前方车队，只能自己形成一个新车队
    - 所以：fleets += 1, current_fleet_time = time

为什么只看到达时间就够了？
- 因为两辆车同向行驶，且不能超过
- 假设后车到终点时间更短：后车更快，那它一定会在终点前或终点处追上前车，所以合并成一个车队
- 假设后车到终点时间更长：后车更慢，那它追不上前车，只能单独成为一个车队
- 所以比较到达时间就可以判断是否合并

```js
/**
 * @param {number} target
 * @param {number[]} position
 * @param {number[]} speed
 * @return {number}
 */
var carFleet = function(target, position, speed) {
    const cars = [];

    for (let i = 0; i < position.length; i++) {
        cars.push([position[i], speed[i]]);
    }

    cars.sort((a, b) => a[0] - b[0]);

    let fleets = 0;
    let currentFleetTime = 0;

    for (let i = cars.length - 1; i >= 0; i--) {
        const p = cars[i][0];
        const s = cars[i][1];

        const time = (target - p) / s;

        if (time > currentFleetTime) {
            fleets++;
            currentFleetTime = time;
        }
    }

    return fleets;
};
```
```python
class Solution:
    def carFleet(self, target: int, position: List[int], speed: List[int]) -> int:
        cars = sorted(zip(position, speed))

        fleets = 0
        current_fleet_time = 0

        for p, s in reversed(cars):
            time = (target - p) / s

            if time > current_fleet_time:
                fleets += 1
                current_fleet_time = time

        return fleets
```
```java
import java.util.*;

class Solution {
    public int carFleet(int target, int[] position, int[] speed) {
        int n = position.length;

        int[][] cars = new int[n][2];

        for (int i = 0; i < n; i++) {
            cars[i][0] = position[i];
            cars[i][1] = speed[i];
        }

        Arrays.sort(cars, (a, b) -> a[0] - b[0]);

        int fleets = 0;
        double currentFleetTime = 0.0;

        for (int i = n - 1; i >= 0; i--) {
            int p = cars[i][0];
            int s = cars[i][1];

            double time = (double) (target - p) / s;

            if (time > currentFleetTime) {
                fleets++;
                currentFleetTime = time;
            }
        }

        return fleets;
    }
}
```

- 时间复杂度：`O(n log n)`
  - 排序需要：`O(n log n)`
  - 之后从右往左扫描一遍：`O(n)`
  - 所以总时间复杂度是：`O(n log n)`
- 空间复杂度：`O(n)`
  - 我们创建了一个 `cars` 数组保存：`[position, speed]`，所以空间复杂度是：`O(n)`
  - 如果不计算排序和输入配对数组的空间，扫描本身只用：`O(1)`
  - 但在 `Python / JS / Java `这种写法里，一般说：`O(n)`
