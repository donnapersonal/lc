# 1482.制作 m 束花所需的最少天数

题目链接：[1482.制作 m 束花所需的最少天数](https://leetcode.cn/problems/minimum-number-of-days-to-make-m-bouquets/description/)

## 题目大意

给你一个整数数组 bloomDay，以及两个整数 m 和 k 

现需要制作 m 束花。制作花束时，需要使用花园中 相邻的 k 朵花 

花园中有 n 朵花，第 i 朵花会在 bloomDay[i] 时盛开，恰好 可以用于 一束 花中

请你返回从花园中摘 m 束花需要等待的最少的天数。如果不能摘到 m 束花则返回 -1 

```js
Example 1:
Input: bloomDay = [1,10,3,10,2], m = 3, k = 1
Output: 3
Explanation: Let us see what happened in the first three days. x means flower bloomed and _ means flower did not bloom in the garden.
We need 3 bouquets each should contain 1 flower.
After day 1: [x, _, _, _, _]   // we can only make one bouquet.
After day 2: [x, _, _, _, x]   // we can only make two bouquets.
After day 3: [x, _, x, _, x]   // we can make 3 bouquets. The answer is 3.

Example 2:
Input: bloomDay = [1,10,3,10,2], m = 3, k = 2
Output: -1
Explanation: We need 3 bouquets each has 2 flowers, that means we need 6 flowers. We only have 5 flowers so it is impossible to get the needed bouquets and we return -1.

Example 3:
Input: bloomDay = [7,7,7,7,12,7,7], m = 2, k = 3
Output: 12
Explanation: We need 2 bouquets each should have 3 flowers.
Here is the garden after the 7 and 12 days:
After day 7: [x, x, x, x, _, x, x]
We can make one bouquet of the first three flowers that bloomed. We cannot make another bouquet from the last three flowers that bloomed because they are not adjacent.
After day 12: [x, x, x, x, x, x, x]
It is obvious that we can make two bouquets in different ways.
```

限制：
- bloomDay.length == n
- 1 <= n <= 10^5
- 1 <= bloomDay[i] <= 10^9
- 1 <= m <= 10^6
- 1 <= k <= n

## 解题

每束花需要 `k` 朵花，一共要做 `m` 束花，所以至少需要：`m * k` 朵花。如果花园里的花总数不够：`bloomDay.length < m * k`，那么无论等多少天，花的数量都不够，直接返回：`-1`

我们要找的是：`最少需要多少天，才能做出 m 束花`

假设有一个函数：`canMake(days) 表示在 days 天内，能不能做出至少 m 束花`

那么这个函数具有`单调性`：
- 很小天数，不能做出 m 束花
- 天数慢慢变大，可能还是不能做出 m 束花
- 达到某一天，可以做出 m 束花
- 更大的天数，一定也可以做出 m 束花

也就是说：`false false false true true true`

一旦某一天可以做出 `m` 束花，那么之后更大的天数也一定可以。因为天数越多，开放的花只会更多，不会变少

所以这就是一个典型的：在单调答案空间中找第一个 `true` -> 可以用`二分查找`

`二分查找的范围`
- 最少天数不可能小于：`bloomDay` 中的最小值，因为至少要等到某些花开放
- 最大天数不可能大于：`bloomDay` 中的最大值，因为到了最大开放日，所有花都已经开放

所以二分范围是：
- `low = min(bloomDay)`
- `high = max(bloomDay)`

`辅助函数 canMake(days) `
- 对于一个给定天数 `days`，需要判断：到第 `days` 天为止，能不能做出至少 `m` 束花
- 规则是：
  - 如果 `bloomDay[i] <= days`，说明这朵花已经开放，可以使用
  - 如果连续开放的花数量达到 `k`，就可以做出一束花
  - 做完一束之后，这 `k` 朵花已经被使用，不能重复使用，所以连续计数要清零
  - 如果遇到一朵还没开放的花，连续段断开，也要清零

`核心贪心逻辑`
- 对于固定的 `days`，从左到右扫描 `bloomDay`
- 维护两个变量：
  - `bouquets` -> 表示已经做出的花束数量
  - `consecutive` -> 表示当前连续开放的花的数量
- 扫描过程：
  - 如果 `bloomDay[i] <= days: consecutive++`
  - 如果 `consecutive == k: bouquets++ consecutive = 0`
  - 否则: `consecutive = 0`

> 为什么做完一束后要把 `consecutive` 清零？
> - 因为每朵花只能用一次
> - 例如 `k = 3`，连续开放了 `3` 朵花：`[open, open, open]`
> - 这 `3` 朵花已经组成一束，不能再参与下一束，所以要重新开始计数

```js
/**
 * @param {number[]} bloomDay
 * @param {number} m
 * @param {number} k
 * @return {number}
 */
var minDays = function(bloomDay, m, k) {
    const n = bloomDay.length;

    // Not enough flowers
    if (m > Math.floor(n / k)) {
        return -1;
    }

    let low = Math.min(...bloomDay);
    let high = Math.max(...bloomDay);

    const canMake = (days) => {
        let bouquets = 0;
        let consecutive = 0;

        for (const day of bloomDay) {
            if (day <= days) {
                consecutive++;

                if (consecutive === k) {
                    bouquets++;
                    consecutive = 0;

                    if (bouquets === m) {
                        return true;
                    }
                }
            } else {
                consecutive = 0;
            }
        }

        return false;
    };

    while (low < high) {
        const mid = Math.floor(low + (high - low) / 2);

        if (canMake(mid)) {
            high = mid;
        } else {
            low = mid + 1;
        }
    }

    return low;
};
```

from typing import List

class Solution:
    def minDays(self, bloomDay: List[int], m: int, k: int) -> int:

        # Not enough flowers

            consecutive = 0

                    consecutive += 1

                    if consecutive == k:
                        bouquets += 1
                        consecutive = 0

                        if bouquets == m:
                            return True
                else:
                    consecutive = 0

            return False

        while low < high:
            mid = low + (high - low) // 2

            if can_make(mid):
                high = mid
            else:
                low = mid + 1

        return low

```java
class Solution {
    public int minDays(int[] bloomDay, int m, int k) {
        int n = bloomDay.length;

        // 如果花的总数不够，直接返回 -1
        // 使用除法避免 m * k 溢出
        if (m > n / k) {
            return -1;
        }

        int low = Integer.MAX_VALUE;
        int high = 0;

        // 找到二分查找的左右边界
        for (int day : bloomDay) {
            low = Math.min(low, day);
            high = Math.max(high, day);
        }

        // 二分查找最小可行天数
        while (low < high) {
            int mid = low + (high - low) / 2;

            if (canMake(bloomDay, mid, m, k)) {
                // mid 天可以做出 m 束花，尝试更小的天数
                high = mid;
            } else {
                // mid 天不够，需要更多天
                low = mid + 1;
            }
        }

        return low;
    }

    private boolean canMake(int[] bloomDay, int days, int m, int k) {
        int bouquets = 0;
        int consecutive = 0;

        for (int day : bloomDay) {
            if (day <= days) {
                consecutive++;

                if (consecutive == k) {
                    bouquets++;
                    consecutive = 0;

                    if (bouquets == m) {
                        return true;
                    }
                }
            } else {
                consecutive = 0;
            }
        }

        return false;
    }
}
```

- 时间复杂度：O(nlog(high−low))，其中 n 是数组 bloomDay 的长度，high 和 low 分别是数组 bloomDay 中的最大值和最小值。
需要遍历数组 bloomDay 得到其中的最大值 high 和最小值 low，遍历的时间复杂度是 O(n)。
得到最大值 high 和最小值 low 之后，二分查找的迭代次数是 O(log(high−low))，每次判断是否能制作规定数量的花束的时间复杂度是 O(n)，因此二分查找的总时间复杂度是 O(nlog(high−low))。
整个算法的时间复杂度是 O(n)+O(nlog(high−low))=O(nlog(high−low))。
- 空间复杂度：O(1)
