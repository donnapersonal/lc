# 2115.从给定原材料中找到所有可以做出的菜

题目链接：[2115.从给定原材料中找到所有可以做出的菜](https://leetcode.cn/problems/find-all-possible-recipes-from-given-supplies/description/)

## 题目大意

你有 n 道不同菜的信息。给你一个字符串数组 recipes 和一个二维字符串数组 ingredients 。第 i 道菜的名字为 recipes[i] ，如果你有它 所有 的原材料 ingredients[i]，那么你可以 做出 这道菜。一份食谱也可以是 其它 食谱的原料，也就是说 ingredients[i] 可能包含 recipes 中另一个字符串

同时给你一个字符串数组 supplies ，它包含你初始时拥有的所有原材料，每一种原材料你都有无限多

请你返回你可以做出的所有菜。你可以以 任意顺序 返回它们

注意两道菜在它们的原材料中可能互相包含

```js
Example 1:
Input: recipes = ["bread"], ingredients = [["yeast","flour"]], supplies = ["yeast","flour","corn"]
Output: ["bread"]
Explanation:
We can create "bread" since we have the ingredients "yeast" and "flour".

Example 2:
Input: recipes = ["bread","sandwich"], ingredients = [["yeast","flour"],["bread","meat"]], supplies = ["yeast","flour","meat"]
Output: ["bread","sandwich"]
Explanation:
We can create "bread" since we have the ingredients "yeast" and "flour".
We can create "sandwich" since we have the ingredient "meat" and can create the ingredient "bread".

Example 3:
Input: recipes = ["bread","sandwich","burger"], ingredients = [["yeast","flour"],["bread","meat"],["sandwich","meat","bread"]], supplies = ["yeast","flour","meat"]
Output: ["bread","sandwich","burger"]
Explanation:
We can create "bread" since we have the ingredients "yeast" and "flour".
We can create "sandwich" since we have the ingredient "meat" and can create the ingredient "bread".
We can create "burger" since we have the ingredient "meat" and can create the ingredients "bread" and "sandwich".
```

限制：
- n == recipes.length == ingredients.length
- 1 <= n <= 100
- 1 <= ingredients[i].length, supplies.length <= 100
- 1 <= recipes[i].length, ingredients[i][j].length, supplies[k].length <= 10
- recipes[i], ingredients[i][j], and supplies[k] consist only of lowercase English letters.
- All the values of recipes and supplies combined are unique.
- Each ingredients[i] does not contain any duplicate values.

## 解题

```js
/**
 * @param {string[]} recipes
 * @param {string[][]} ingredients
 * @param {string[]} supplies
 * @return {string[]}
 */
var findAllRecipes = function(recipes, ingredients, supplies) {
    const graph = new Map();
    const indegree = new Map();

    // Build graph: ingredient -> recipes that need this ingredient
    for (let i = 0; i < recipes.length; i++) {
        const recipe = recipes[i];
        const ingredientList = ingredients[i];

        indegree.set(recipe, ingredientList.length);

        for (const ingredient of ingredientList) {
            if (!graph.has(ingredient)) {
                graph.set(ingredient, []);
            }
            graph.get(ingredient).push(recipe);
        }
    }

    const res = [];
    const queue = [...supplies];
    let head = 0;

    // BFS / Topological Sort
    while (head < queue.length) {
        const cur = queue[head++];

        if (!graph.has(cur)) {
            continue;
        }

        for (const recipe of graph.get(cur)) {
            indegree.set(recipe, indegree.get(recipe) - 1);

            if (indegree.get(recipe) === 0) {
                res.push(recipe);
                queue.push(recipe);
            }
        }
    }

    return res;
};
```
```python
class Solution:
    def findAllRecipes(self, recipes: List[str], ingredients: List[List[str]], supplies: List[str]) -> List[str]:
        graph = defaultdict(list)
        deg = {}
        for r, ing in zip(recipes, ingredients):
            for s in ing:
                graph[s].append(r)  # 从这道菜的原材料向这道菜连边
            deg[r] = len(ing)
        res = []
        q = deque(supplies)  # 拓扑排序（用初始原材料当队列）
        while q:
            for r in graph[q.popleft()]:
                deg[r] -= 1
                if deg[r] == 0:  # 这道菜的所有原材料我们都有
                    q.append(r)
                    res.append(r)
        return res
```

```java
class Solution {
    public List<String> findAllRecipes(String[] recipes, List<List<String>> ingredients, String[] supplies) {
        Map<String, List<String>> graph = new HashMap<>();
        Map<String, Integer> missing = new HashMap<>();

        for (int i = 0; i < recipes.length; i++) {
            String recipe = recipes[i];
            List<String> ingredientList = ingredients.get(i);

            missing.put(recipe, ingredientList.size());

            for (String ingredient : ingredientList) {
                graph
                    .computeIfAbsent(ingredient, k -> new ArrayList<>())
                    .add(recipe);
            }
        }

        List<String> res = new ArrayList<>();
        Queue<String> queue = new ArrayDeque<>();

        for (String supply : supplies) {
            queue.offer(supply);
        }

        while (!queue.isEmpty()) {
            String item = queue.poll();

            if (!graph.containsKey(item)) {
                continue;
            }

            for (String recipe : graph.get(item)) {
                int newMissing = missing.get(recipe) - 1;
                missing.put(recipe, newMissing);

                if (newMissing == 0) {
                    res.add(recipe);
                    queue.offer(recipe);
                }
            }
        }

        return res;
    }
}
```

- Time Complexity: `O(n + m + E)`, where E is the total number of ingredient dependencies.
If each recipe has at most d ingredients, then E <= dn, so it can be simplified to O(dn + m).
- Space Complexity: `O(n + m + E)`, or O(dn + m) under the same assumption.
