## 解题

Restate
> - Input: 
>   - sorted nums1 has m+n length, with the first m elements being valid, the last n positions of nums1 are used to store the merged result
>   - sorted nums2 has n sorted elements
> - Output: No return a new array, merge nums1 and nums2 in increasing order, store the result in nuns1
> - Clarifications：
>   - Are both arrays sorted in non-decreasing order?
>   - Should I modify nums1 in place?
>   - Can the arrays contain duplicates and negative numbers?
>   - Are the trailing zeros in nums1 only placeholders?

Recognition
> Keywords: 
> - two sorted arrays
> - merge
> - in place
> - nums1 has extra space at the end
> - non-decreasing
> 
> Pattern: 
> - Two pointers
> - Merge from right to left
> 
> 为什么适用
> - 两个数组已经有序，可以使用双指针比较当前元素
> - 由于 nums1 尾部有空位，如果从后向前放置较大的元素，就不会覆盖 nums1 中尚未处理的数据

Brute Force
> `Two Pointers + extra []`
> - define temporary sorted = [], and p1 -> nums1, p2 -> nums2
> - compare 2 elements, append smaller element into sorted
> - add the remaining elements to sorted
> - copy sorted back to nums1
>
> Time: `O(m+n)`, Space: `O(m+n)`
> - Scan 2 arrays: `O(m+n)`
> - copy back: `O(m+n)`
> - sorted array stores all m+n elements，space is `O(m+n)`
>
> Bottleneck:
> - This solution involves no redundant calculations, and its time complexity is already optimal.
> - The extra result array requires `O(m+n)` space

```python
class Solution:
    def merge(self, nums1: List[int], m: int, nums2: List[int], n: int) -> None:
        sorted = []
        p1 = 0
        p2 = 0

        while p1 < m and p2 < n:
            if nums1[p1] <= nums2[p2]:
                sorted.append(nums1[p1])
                p1 += 1
            else:
                sorted.append(nums2[p2])
                p2 += 1

        while p1 < m:
            sorted.append(nums1[p1])
            p1 += 1

        while p2 < n:
            sorted.append(nums2[p2])
            p2 += 1

        nums1[:] = sorted
```

Optimization
> - no need to create a new array, can fill nums1 from right to left
> - nums1 already has empty space at the end
> - because both sorted, largest elements come last; we can start from the end of both arrays and place the largest value at the end of nums1
> - this avoid overwriting unprocessed nums1 elements

Solution:
> - using 3 pointers: 
>   - P1 = `m-1 `-> nums1 last valid element
>   - p2 = `n-1` -> nums2 last element
>   - k = `m+n-1` -> nums1 last position
> - each time, compare `nums1[p1]` and `nums2[p2]`, place the larger one in `nums1[k]`, move relative pointer
> - move `k` pointer
> - repeat until finishing `nums2`
>
> 为什么只检查 `p2 >= 0`?
> - 如果 `nums2` 已经处理完，`nums1` 剩余元素本来就在正确位置
> - 如果 `nums1` 先处理完，仍然需要把 `nums2` 剩余元素复制进来
> 
> 两个数组未处理部分的最大元素一定是：`max(nums1[p1], nums2[p2])`，把它放到当前最后的空位 `k`，这个位置必然正确，不断重复，最终整个数组有序

```python
class Solution:
    def merge(self, nums1: List[int], m: int, nums2: List[int], n: int) -> None:
        # p1 指向 nums1 最后一个有效元素
        # p2 指向 nums2 最后一个元素
        # k 指向 nums1 最后一个可以写入的位置
        p1, p2, k = m-1, n-1, m+n-1

        # 若两个数组都有元素
        while p1 >= 0 and p2 >= 0:
            if nums1[p1] > nums2[p2]:
                nums1[k] = nums1[p1]
                p1 -= 1
            else:
                nums1[k] = nums2[p2]
                p2 -= 1
            
            k -= 1
        
        # 判断 nums2, 如果 `nums1` 先处理完，仍然需要把 `nums2` 剩余元素复制进来
        while p2 >= 0:
            nums1[k] = nums2[p2]
            p2 -= 1
            k -= 1
```
```java
class Solution {
    public void merge(int[] nums1, int m, int[] nums2, int n) {
        int p1 = m-1;
        int p2 = n-1;
        int k = m+n - 1;
        while (p1 >= 0 && p2 >= 0) {
            if (nums1[p1] > nums2[p2]) {
                nums1[k] = nums1[p1];
                p1 --;
            } else {
                nums1[k] = nums2[p2];
                p2 --;
            }
            k --;
        }
        while (p2 >= 0) {
            nums1[k] = nums2[p2];
            p2 -= 1;
            k -= 1;
        }
    }
}
```

Complexity
> - Time: `O(m + n)`, in worst case, each element is processed at most once
> - Space: `O(1)`, just use 3 pointers

Variants
- 1. if nums1 has no extra space
> - cannot modify in-place
> - use extra array to store
> - Time: O(m + n), Space: O(m + n)

- 2. if two arrays are not sorted
> - two pointers don't work, the last element is no longer the maximum value
> - can merge first and then sort
> - Time: O((m+n)log(m+n))

- 3. if need return new array
> - use two pointers, put the result in res, return res

- 4. if return original index
> - need to record: `(value, array_id, original_index)`

- 5. if need to deduplicate element
> - check if the current value equals to the previous written value
> - or using two pointers to create new array

- 6. if has k sorted arrays
> - k 个有序序列，需要重复获得全局最小元素 → Min Heap
> - 核心原则是：最小堆中不保存所有元素，只保存每个数组/数据流“当前最小的候选元素”，这样堆的大小最多是 k
```python
例如：
arrays = [
    [1, 4, 7],
    [2, 5, 8],
    [3, 6, 9]
]

初始化时，把每个数组的第一个元素放进最小堆：

(1, array 0, index 0)
(2, array 1, index 0)
(3, array 2, index 0)

每次：
- 弹出堆中最小元素
- 加入结果
- 从该元素所属的数组中取下一个元素
- 将下一个元素加入堆
import heapq
from typing import List


def merge_k_sorted_arrays(arrays: List[List[int]]) -> List[int]:
    min_heap = []
    res = []

    # 每个非空数组的第一个元素进入堆
    for array_id, array in enumerate(arrays):
      if array:
        # (value, array_id, index)
        # - value：当前元素值，用于最小堆排序
        # - array_id：元素来自哪个数组
        # - index：元素在该数组中的位置。
        # 必须记录来源，弹出元素后才能找到同一个数组中的下一个元素
        heapq.heappush(min_heap, (array[0], array_id, 0))
    
    while min_heap:
      value, array_id, index = heapq.heappop(min_heap)
      res.append(value)
      next_index = index + 1
      if next_index < len(arrays[array_id]):
        next_value = arrays[array_id][next_index]
        heapq.heappush(min_heap, (next_value, array_id, next_index))
    
    return res

# N 是所有数组的元素总数；k 是数组数量
# 每个元素：进入堆一次，离开堆一次。堆最多有 k 个元素
# Time: O(N log k)
# Space: Output Space: O(N), Heap Space: O(k)
# 通常复杂度分析会将结果数组视为必要输出，因此说额外空间是 O(k)
```

- 7. if input is streaming
> - 数据流与数组的区别是：
>   - 数组可以通过下标访问
>   - 数据流通常只能不断调用 next() 获取下一个元素
> 
> 数据流可能非常大，甚至无限，不能一次全部读入内存。对于两个有序流，只需要分别保存两个流的当前元素

```python
from typing import Iterator
def merge_two_sorted_streams(stream1: Iterator[int], stream2: Iterator[int]) -> Iterator[int]:
    end = object()

    value1 = next(stream1, end)
    value2 = next(stream2, end)

    while value1 is not end and value2 is not end:
        if value1 <= value2:
            yield value1
            value1 = next(stream1, end)
        else:
            yield value2
            value2 = next(stream2, end)

    while value1 is not end:
        yield value1
        value1 = next(stream1, end)

    while value2 is not end:
        yield value2
        value2 = next(stream2, end)

# 使用方法
stream1 = iter([1, 4, 7])
stream2 = iter([2, 3, 8])

merged = merge_two_sorted_streams(stream1, stream2)

for value in merged:
    print(value)

# 使用 yield 的好处是不会一次创建完整结果数组。每请求一个结果，我们才生成一个结果
# Invariant: value1 和 value2 分别是两个流中尚未输出的最小元素，因此二者中较小的元素就是当前全局最小元素
# 复杂度
# 如果总共输出 N 个元素：
# Time: O(N)
# Extra Space: O(1)

# 如果两个流是无限流，复杂度通常按每个输出元素计算：
# Time per output: O(1)
# Space: O(1)
```
