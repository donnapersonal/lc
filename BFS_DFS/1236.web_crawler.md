# 1236.网络爬虫

题目链接：[1236.网络爬虫](https://leetcode.cn/problems/web-crawler/)

## 题目大意

给定一个网址 startUrl 和一个接口 HtmlParser ，请你实现一个网络爬虫，以实现爬取同 startUrl 拥有相同 域名 的全部链接

该爬虫得到的全部网址可以 任何顺序 返回结果

你的网络爬虫应当按照如下模式工作：
- 自页面 startUrl 开始爬取
- 调用 HtmlParser.getUrls(url) 来获得给定 url 网址中的全部链接
- 同一个链接最多只爬取一次
- 只浏览 域名 与 startUrl 相同 的链接集合

限制：
- 1 <= urls.length <= 1000
- 1 <= urls[i].length <= 300
- startUrl is one of the urls.
- Hostname label must be from 1 to 63 characters long, including the dots, may contain only the ASCII letters from 'a' to 'z', digits  from '0' to '9' and the hyphen-minus character ('-').
- The hostname may not start or end with the hyphen-minus character ('-').

## 解题

深度优先搜索（DFS）是一种遍历图的算法，适用于有向图和无向图。本题中处理的是一张有向图，而且图并没有显式给出。对于某个顶点对应的 URL，需要调用 htmlParser.getUrls(url) 才能得到它的出边

用递归方式实现 DFS

递归函数 dfs 的参数是当前正在访问的顶点 url。

从 url 出发，可能有若干条出边通向其他顶点。我们通过 htmlParser.getUrls(url) 枚举所有这样的顶点 nextUrl。如果 nextUrl 还没有访问过，且它的域名与起始 URL 的域名相同，就递归调用 dfs 访问它。访问 nextUrl 时，算法会继续处理它的所有出边。这样就能遍历到所有需要访问的顶点

因为只有在相邻顶点尚未访问时才继续递归，所以任何顶点都不会被访问两次。我们可以用集合 visited 记录已经访问过的 URL。DFS 结束后，visited 中保存的就是答案

算法步骤

递归函数 dfs 接收当前正在访问的 url。同时维护哈希集合 visited，记录所有已经被 dfs 访问过的 URL

dfs(url) 的过程如下：
- 将 url 加入哈希集合 visited
- 遍历 htmlParser.getUrls(url) 返回的所有 URL。记当前枚举到的元素为 nextUrl
- 如果 nextUrl 的域名与起始 URL 的域名相同，并且 visited 中还没有 nextUrl，就递归调用 dfs(nextUrl)
  
主函数中，先求出 startUrl 的域名 startHostname，然后从 startUrl 开始执行 DFS，最后返回集合 visited 中的所有 URL

/**
 * // This is the HtmlParser's API interface.
 * // You should not implement it, or speculate about its implementation
 * function HtmlParser() {
 *
 *		@param {string} url
 *     	@return {string[]}
 *     	this.getUrls = function(url) {
 *      	...
 *     	};
 * };
 */

/**
 * @param {string} startUrl
 * @param {HtmlParser} htmlParser
 * @return {string[]}
*/
var crawl = function(startUrl, htmlParser) {
    function getHostname(url) {
        // Example:
        // "http://example.org/foo/bar"
        // split("/") -> ["http:", "", "example.org", "foo", "bar"]
        return url.split("/")[2];
    }

    const startHostname = getHostname(startUrl);
    const queue = [startUrl];
    const visited = new Set([startUrl]);

    let head = 0;

    while (head < queue.length) {
        const url = queue[head++];

        const nextUrls = htmlParser.getUrls(url);

        for (const nextUrl of nextUrls) {
            if (
                getHostname(nextUrl) === startHostname &&
                !visited.has(nextUrl)
            ) {
                visited.add(nextUrl);
                queue.push(nextUrl);
            }
        }
    }

    return Array.from(visited);
};

/**
 * // This is the HtmlParser's API interface.
 * // You should not implement it, or speculate about its implementation
 * interface HtmlParser {
 *     public List<String> getUrls(String url) {}
 * }
 */

class Solution {
    public List<String> crawl(String startUrl, HtmlParser htmlParser) {
        String startHostname = getHostname(startUrl);

        Queue<String> queue = new LinkedList<>();
        Set<String> visited = new HashSet<>();

        queue.offer(startUrl);
        visited.add(startUrl);

        while (!queue.isEmpty()) {
            String url = queue.poll();

            for (String nextUrl : htmlParser.getUrls(url)) {
                if (
                    getHostname(nextUrl).equals(startHostname) &&
                    !visited.contains(nextUrl)
                ) {
                    visited.add(nextUrl);
                    queue.offer(nextUrl);
                }
            }
        }

- 时间复杂度：`O(m⋅l)`
  - 设 m 为图中的边数，l 为 URL 的最大长度（即 urls[i].length）
  - 设 k 为实际遍历到的顶点数。将这些顶点加入集合时，每个顶点的代价最多为 O(l)，因此总插入代价为 O(k⋅l)
  - 在 dfs 中，最耗时的部分是调用 htmlParser.getUrls(url) 得到从 url 出发的所有边，并遍历其中的每个 nextUrl。处理 nextUrl 时，需要提取它的域名并在哈希集合中查询它，这两步都可能花费 O(nextUrl.length)=O(l) 时间
  - 所有 htmlParser.getUrls(url) 返回列表中的元素总数就是图的总边数 m。而每个元素的长度最多为 O(l)，因此这部分总代价为 `O(m⋅l)`
  - 总复杂度是 O(k⋅l+m⋅l)。由于 k=O(m)，可以化简为 `O(m⋅l)`
- 空间复杂度：`O(m⋅l)`
  - 在递归的每一层，都会暂存 htmlParser.getUrls(url) 的返回值。前面已经说明过，这些字符串总长度为 O(m⋅l)
  - 此外，还要用集合保存答案，这个集合也可能增长到同样的规模。虽然通常不会把答案本身计入空间复杂度，但这里的集合同时还承担了功能性作用，即防止同一个 URL 被重复访问
