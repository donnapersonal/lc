# 756.金字塔转换矩阵

题目链接：[756.金字塔转换矩阵](https://leetcode.cn/problems/pyramid-transition-matrix/)

## 题目大意

你正在把积木堆成金字塔。每个块都有一个颜色，用一个字母表示。每一行的块比它下面的行 少一个块 ，并且居中

为了使金字塔美观，只有特定的 三角形图案 是允许的。一个三角形的图案由 两个块 和叠在上面的 单个块 组成。模式是以三个字母字符串的列表形式 allowed 给出的，其中模式的前两个字符分别表示左右底部块，第三个字符表示顶部块

例如，"ABC" 表示一个三角形图案，其中一个 “C” 块堆叠在一个 'A' 块(左)和一个 'B' 块(右)之上。请注意，这与 "BAC" 不同，"B" 在左下角，"A" 在右下角

你从作为单个字符串给出的底部的一排积木 bottom 开始，必须 将其作为金字塔的底部

在给定 bottom 和 allowed 的情况下，如果你能一直构建到金字塔顶部，使金字塔中的 每个三角形图案 都是在 allowed 中的，则返回 true ，否则返回 false

限制：
- 2 <= bottom.length <= 6
- 0 <= allowed.length <= 216
- allowed[i].length == 3
- The letters in all input strings are from the set {'A', 'B', 'C', 'D', 'E', 'F'}.
- All the values of allowed are unique.

## 解题

这题可以理解成：已知最底下一层 bottom，以及一些规则 allowed，每条规则表示：下面相邻两个块可以支撑上面一个块。
问能不能一层一层往上搭，最后搭成只剩一个块的金字塔

例如规则：ABC 表示：
A B
 C

也就是下面相邻两个字符是 A 和 B 时，上面可以放 C

推荐方法：`DFS + 回溯 + 记忆化`
核心思想: 
- 把 当前这一层 row 当成一个状态
- 比如：bottom = "BCD"，如果规则允许：BC -> A / E，CD -> F，那么下一层可能是：
  AF
  EF
- 然后继续对 "AF" 或 "EF" 做同样的事情，直到某一层长度变成 1
- 如果能走到长度为 1，说明可以成功搭成金字塔

第一步：建立规则表
- 把 allowed 转换成一个映射：graph[(a, b)] = [c1, c2, c3]
  - 意思是：如果下面相邻两个块是 a 和 b，那么上面可以放哪些字符
  - 例如：allowed = ["ABC", "ABD", "BCE"] 可以转成：(A, B) -> [C, D]，(B, C) -> [E]
  - 这样后面查规则会很方便

第二步：递归判断当前 row 能不能成功
- 定义：dfs(row) 表示当前这一层是 row，能不能继续往上搭到顶部？
- 如果：len(row) == 1，说明已经搭到顶部，直接返回 True

第三步：用回溯构造下一层
- 假设当前层是：row = "ABCD"，那么下一层长度应该是 3，因为：
  > A B C D
  >  ? ? ?
- 下一层第一个字符由 A B 决定
- 下一层第二个字符由 B C 决定
- 下一层第三个字符由 C D 决定
- 也就是说：row[i] 和 row[i + 1] 决定 next_row[i]
- 以用回溯一位一位构造 next_row

> 为什么需要记忆化？
> - 因为不同路径可能生成同一个中间层
> - 例如：bottom -> row1 -> "AB", bottom -> row2 -> "AB"
> - 如果 "AB" 已经尝试过，并且发现它无法成功搭到顶部，那么下次再遇到 "AB" 就不用重复算了
> - 所以我们用：@lru_cache(None)
> - 缓存每个 row 的结果，这可以避免大量重复搜索

```js
/**
 * @param {string} bottom
 * @param {string[]} allowed
 * @return {boolean}
 */
var pyramidTransition = function(bottom, allowed) {
    const graph = new Map();

    // Build transition map:
    // key: "AB", value: ["C", "D", ...]
    for (const pattern of allowed) {
        const a = pattern[0];
        const b = pattern[1];
        const c = pattern[2];

        const key = a + b;

        if (!graph.has(key)) {
            graph.set(key, []);
        }

        graph.get(key).push(c);
    }

    const memo = new Map();

    function dfs(row) {
        // Successfully built to the top
        if (row.length === 1) {
            return true;
        }

        // If this row has already been computed
        if (memo.has(row)) {
            return memo.get(row);
        }

        function buildNext(index, path) {
            // Finished building one possible next row
            if (index === row.length - 1) {
                const nextRow = path.join('');
                return dfs(nextRow);
            }

            const key = row[index] + row[index + 1];

            // No valid upper block for this adjacent pair
            if (!graph.has(key)) {
                return false;
            }

            for (const top of graph.get(key)) {
                path.push(top);

                if (buildNext(index + 1, path)) {
                    return true;
                }

                path.pop();
            }

            return false;
        }

        const result = buildNext(0, []);
        memo.set(row, result);
        return result;
    }

    return dfs(bottom);
};
```
```python
from typing import List
from collections import defaultdict
from functools import lru_cache

class Solution:
    def pyramidTransition(self, bottom: str, allowed: List[str]) -> bool:
        graph = defaultdict(list)

        for a, b, c in allowed:
            graph[(a, b)].append(c)

        @lru_cache(None)
        def dfs(row: str) -> bool:
            if len(row) == 1:
                return True

            def build_next(i: int, path: list) -> bool:
                if i == len(row) - 1:
                    return dfs(''.join(path))

                pair = (row[i], row[i + 1])
                if pair not in graph:
                    return False

                for top in graph[pair]:
                    path.append(top)
                    if build_next(i + 1, path):
                        return True
                    path.pop()

                return False

            return build_next(0, [])

        return dfs(bottom)
```

```java
import java.util.*;

class Solution {
    private Map<String, List<Character>> graph;
    private Map<String, Boolean> memo;

    public boolean pyramidTransition(String bottom, List<String> allowed) {
        graph = new HashMap<>();
        memo = new HashMap<>();

        // Build transition map:
        // key: "AB", value: ['C', 'D', ...]
        for (String pattern : allowed) {
            char a = pattern.charAt(0);
            char b = pattern.charAt(1);
            char c = pattern.charAt(2);

            String key = "" + a + b;

            graph.putIfAbsent(key, new ArrayList<>());
            graph.get(key).add(c);
        }

        return dfs(bottom);
    }

    private boolean dfs(String row) {
        // Successfully built to the top
        if (row.length() == 1) {
            return true;
        }

        // If this row has already been computed
        if (memo.containsKey(row)) {
            return memo.get(row);
        }

        boolean result = buildNext(row, 0, new StringBuilder());

        memo.put(row, result);
        return result;
    }

    private boolean buildNext(String row, int index, StringBuilder path) {
        // Finished building one possible next row
        if (index == row.length() - 1) {
            return dfs(path.toString());
        }

        String key = "" + row.charAt(index) + row.charAt(index + 1);

        // No valid upper block for this adjacent pair
        if (!graph.containsKey(key)) {
            return false;
        }

        for (char top : graph.get(key)) {
            path.append(top);

            if (buildNext(row, index + 1, path)) {
                return true;
            }

            path.deleteCharAt(path.length() - 1);
        }

        return false;
    }
}
```

- Time Complexity:
  - Worst case is exponential. If N is the length of bottom and A is the alphabet size, a precise upper bound is O(A^(N(N-1)/2)), because there are N(N-1)/2 positions above the bottom row. 
  - A looser explanation is O(A^N) per level / exponential search.

- Space Complexity:
  - For the pyramid-array implementation, O(N^2) for the pyramid storage, plus memoization states.
  - For the string DFS implementation, O(number of memo states + N^2) in the worst case.
